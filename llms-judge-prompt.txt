## Evaluation dimensions

### 1. Framework routing clarity (weight: high)
Can an agent that receives "how do I use Scalekit with LangChain?" immediately identify which URL to fetch without loading the entire AgentKit documentation set?

Score criteria:
- 1–3: No framework-specific routing; agent must load full AgentKit docs
- 4–6: Some routing hints but incomplete or buried
- 7–10: Explicit per-framework URLs visible at the top level; agent can navigate directly

### 2. Product routing precision (weight: high)
For each of these developer intents, does the index clearly point to the right documentation set?

Test intents:
- "I want to add SSO to my SaaS app"
- "I'm building an AI agent that calls Gmail on behalf of users"
- "I need to set up SCIM directory sync"
- "I want machine-to-machine API authentication"
- "How do I get started with Scalekit?"

Score criteria:
- 1–3: Routing is absent or requires reading 50+ lines to find the right set
- 4–6: Routing exists but is ambiguous or lacks clear action
- 7–10: Each intent maps to a specific, named documentation set within the first screenful

### 3. Token efficiency (weight: medium)
Does the structure help an agent load only what it needs, rather than the full 4MB llms-full.txt?

Score criteria:
- 1–3: Only one or two sets available; agent over-fetches for narrow questions
- 4–6: Multiple sets exist but granularity is coarse
- 7–10: Targeted sets exist for common narrow queries (e.g. framework-specific, quickstarts only)

### 4. Page discoverability (weight: medium)
For a developer working on a specific feature, can an agent find the right individual page URL without guessing?

Score criteria:
- 1–3: Pages listed as undifferentiated flat list; no section grouping
- 4–6: Some grouping but product boundaries unclear
- 7–10: Pages grouped by product and sub-topic; easy to scan to the right section

### 5. Coverage completeness (weight: low)
Are all major Scalekit product areas represented? Check for: AgentKit, SaaSKit/FSA, Enterprise SSO, SCIM, MCP Authentication, M2M Auth, SDK/API Reference.

Score criteria:
- 1–3: One or more major product areas missing
- 4–6: All products present but some have thin coverage
- 7–10: All products present with appropriate depth
