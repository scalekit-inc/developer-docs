  <CB.ClassMethod name="actions.tools.list_scoped_tools">
    Lists tools scoped to a specific user. Use this method for tool discovery because it returns pagination metadata such as `next_page_token` and `total_size`; framework `get_tools()` helpers return framework-ready tool objects and do not expose that metadata.

    <CB.ClassMethodInput name="identifier" type="str">
        User connected account identifier **Required.**
      </CB.ClassMethodInput>
    <CB.ClassMethodInput name="filter" type="ScopedToolFilter">
        Filter by providers, tool names, or connection names
      </CB.ClassMethodInput>
    <CB.ClassMethodInput name="page_size" type="int">
        Maximum tools per page.
      </CB.ClassMethodInput>
    <CB.ClassMethodInput name="page_token" type="str">
        Opaque cursor from a previous list response
      </CB.ClassMethodInput>
    <CB.ClassMethodOutput type="ListScopedToolsResponse">
        List scoped tools.
      </CB.ClassMethodOutput>
  </CB.ClassMethod>

```python title="Example"
tools_response = scalekit_client.actions.tools.list_scoped_tools(
    identifier="user@example.com",
    page_size=100,
)
# Pass tools_response.tools to your LLM's tool call API
```
