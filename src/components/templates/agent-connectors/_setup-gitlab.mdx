Register your Scalekit environment with the GitLab connector so Scalekit handles the OAuth 2.0 flow and token lifecycle on your behalf. The connection name you create here is the string you pass to `connection_name` (Python) or `connectionName` (Node.js) in every SDK call.

<Steps>
1. ### Create a connection in Scalekit

    - In [Scalekit dashboard](https://app.scalekit.com), go to **Agent Auth** тЖТ **Connections** and click **Create Connection**.
    - Search for **GitLab** and click **Create**.
    - Click **Use your own credentials** and copy the **Redirect URI**. It looks like:

      `https://<SCALEKIT_ENVIRONMENT_URL>/sso/v1/oauth/<CONNECTION_ID>/callback`

    Keep this tab open тАФ you will return to it in step 4.

    ![](@/assets/docs/agent-connectors/gitlab/use-own-credentials-redirect-uri.png)

2. ### Create a GitLab OAuth application

    - Sign in to your GitLab account and go to **User Settings** тЖТ **Applications**:
      `https://gitlab.com/-/profile/applications`

    - Fill in the form:
      - **Name** тАФ a label for your application (e.g., `My DevOps Agent`)
      - **Redirect URI** тАФ paste the Redirect URI you copied from Scalekit

    - Under **Scopes**, select the permissions your agent needs:

      | Scope | Access granted | Required for |
      | --- | --- | --- |
      | `api` | Full read/write access to all API endpoints (REST + GraphQL) | Most tools |
      | `read_user` | Access to current user profile | `gitlab_current_user_get` |
      | `read_api` | Read-only access to all API endpoints | Read-only agents |
      | `read_repository` | Read access to repositories | File/commit reads only |
      | `write_repository` | Push access to repositories | `gitlab_file_create`, `gitlab_file_update`, `gitlab_file_delete`, `gitlab_branch_create` |

      <Aside type="tip" title="Use api scope for full access">
        The `api` scope grants complete REST and GraphQL access and covers all 110 tools in this connector. Use `read_api` alone if your agent only reads data, or combine `read_user` + `write_repository` for minimal push access without full API access.
      </Aside>

    - Click **Save application**.

    ![](@/assets/docs/agent-connectors/gitlab/create-oauth-app.png)

    <Aside type="note" title="GitLab SaaS vs. self-managed">
      These steps are for **GitLab.com** (SaaS). If your team uses a self-managed GitLab instance, replace `gitlab.com` with your instance hostname in all URLs and in `GITLAB_BASE_URL` when making API calls.
    </Aside>

3. ### Copy your Application ID and Secret

    After saving, GitLab shows the **Application ID** and **Secret** on the application detail page.

    Copy both values тАФ you need them in the next step. The secret is shown only once; if you lose it, regenerate it from the same page.

4. ### Add credentials in Scalekit

    - Return to [Scalekit dashboard](https://app.scalekit.com) тЖТ **Agent Auth** тЖТ **Connections** and open the connection you created in step 1.
    - Enter the following values:
      - **Application ID** тЖТ paste as **Client ID**
      - **Secret** тЖТ paste as **Client Secret**
      - **Scopes** тЖТ enter the same scopes you selected in GitLab (e.g., `api read_user`)
    - Click **Save**.

    ![](@/assets/docs/agent-connectors/gitlab/add-credentials.png)

    <Aside type="note" title="Connection name is your identifier">
      The connection name you set here (e.g., `gitlab`) is the string you pass to `connection_name` in every SDK call. It must match exactly тАФ including case.
    </Aside>
</Steps>
