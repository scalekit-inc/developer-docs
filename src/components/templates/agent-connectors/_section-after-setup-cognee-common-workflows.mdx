<summary>Add data to a dataset</summary>

Add text to a dataset and build its knowledge graph in one call. This is the current workaround for `cognee_remember` (coming soon) — pass the content in `data` and `cognee_improve` ingests and graphs it the same way.

```python
result = actions.execute_tool(
    connection_name='cognee',
    identifier='user_123',
    tool_name="cognee_improve",
    tool_input={
        "data": "Acme Corp signed a 2-year contract on 2026-01-15 for the Enterprise plan.",
        "datasetName": "sales_notes",
    }
)

print(result.result)
```

</details>

<details>
<summary>Recall by meaning</summary>

Search a dataset's knowledge graph with a natural-language question. `cognee_recall` returns a graph-grounded answer by default — use `searchType` to switch retrieval strategies.

```python
result = actions.execute_tool(
    connection_name='cognee',
    identifier='user_123',
    tool_name="cognee_recall",
    tool_input={
        "query": "What plan did Acme Corp sign up for, and when?",
        "datasets": ["sales_notes"],
        "topK": 10,
    }
)

print(result.result.get("answer"))
```

<Aside type="note" title="Search strategies">
  `GRAPH_COMPLETION` (default) returns graph context plus an LLM-generated answer. Use `CHUNKS` or `SUMMARIES` to get raw matching content instead of a synthesized answer, or `RAG_COMPLETION` for classic retrieval-augmented generation.
</Aside>

</details>

<details>
<summary>Consolidate session memory</summary>

Bridge memory cached under a session ID into the permanent knowledge graph, so it becomes searchable by `cognee_recall` outside that session.

```python
result = actions.execute_tool(
    connection_name='cognee',
    identifier='user_123',
    tool_name="cognee_improve",
    tool_input={
        "datasetName": "sales_notes",
        "sessionIds": ["agent-run-1718000000"],
        "runInBackground": True,  # return immediately; graph builds server-side
    }
)

print(result.result)
```

</details>

<details>
<summary>Delete a dataset, an item, or everything</summary>

`cognee_forget` deletes at three levels of scope — a single data item, a whole dataset, or (with `everything`) every dataset on the account. Each level is permanent and cannot be undone.

```python
# Delete a single data item
result = actions.execute_tool(
    connection_name='cognee',
    identifier='user_123',
    tool_name="cognee_forget",
    tool_input={
        "datasetId": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
        "dataId": "7b1c9f0e-2c3d-4e5f-8a9b-0c1d2e3f4a5b",
    }
)
print(result.result)
```

```python
# Delete an entire dataset by name
result = actions.execute_tool(
    connection_name='cognee',
    identifier='user_123',
    tool_name="cognee_forget",
    tool_input={"dataset": "sales_notes"}
)
print(result.result)
```

<Aside type="caution" title="Everything means everything">
  Setting `everything: true` permanently deletes ALL datasets and all memory owned by the account — not just the dataset you're working with. Reserve it for full resets, and never pass it based on unvalidated agent or user input.
</Aside>

</details>

<details>
<summary>End-to-end workflow: improve, recall, forget</summary>

A typical Cognee session adds data, queries it, and cleans up when the dataset is no longer needed.

```python
dataset_name = "support_thread_4821"

# 1. Add data and build the knowledge graph (cognee_remember workaround)
actions.execute_tool(
    connection_name='cognee',
    identifier='user_123',
    tool_name="cognee_improve",
    tool_input={
        "data": "Customer reported a login failure on 2026-02-03. Root cause: expired SSO certificate. Resolved by rotating the certificate.",
        "datasetName": dataset_name,
    }
)

# 2. Recall the relevant context later
recall_result = actions.execute_tool(
    connection_name='cognee',
    identifier='user_123',
    tool_name="cognee_recall",
    tool_input={
        "query": "Why did the customer's login fail, and how was it fixed?",
        "datasets": [dataset_name],
    }
)
print(recall_result.result.get("answer"))

# 3. Forget the dataset once the thread is closed and no longer needed
actions.execute_tool(
    connection_name='cognee',
    identifier='user_123',
    tool_name="cognee_forget",
    tool_input={"dataset": dataset_name}
)
```

</details>

<details>
<summary>LangChain integration</summary>

Let an LLM decide which Cognee tool to call based on natural language. This example builds an agent that can enrich memory, recall it, and clean it up on demand.

```python
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_openai_tools_agent
from langchain_core.prompts import (
    ChatPromptTemplate, SystemMessagePromptTemplate,
    HumanMessagePromptTemplate, MessagesPlaceholder, PromptTemplate
)

# Load all Cognee tools in LangChain format. Use page_size=100 so connector tool lists are not truncated.
tools = actions.langchain.get_tools(
    identifier='user_123',
    providers=["COGNEE"],
    page_size=100
)

prompt = ChatPromptTemplate.from_messages([
    SystemMessagePromptTemplate(prompt=PromptTemplate(
        input_variables=[],
        template=(
            "You are an assistant with access to Cognee memory tools. "
            "Use cognee_improve to add or enrich data in a dataset, cognee_recall to "
            "search stored memory by meaning, and cognee_forget to delete a dataset "
            "or data item. cognee_remember is not available yet — use cognee_improve "
            "with the data field instead. Always confirm before calling cognee_forget."
        )
    )),
    MessagesPlaceholder(variable_name="chat_history", optional=True),
    HumanMessagePromptTemplate(prompt=PromptTemplate(
        input_variables=["input"], template="{input}"
    )),
    MessagesPlaceholder(variable_name="agent_scratchpad")
])

llm = ChatOpenAI(model="gpt-4o")
agent = create_openai_tools_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

result = agent_executor.invoke({
    "input": "Remember that Acme Corp renewed their contract today for the Enterprise plan, then tell me what we know about Acme Corp."
})
print(result["output"])
```
