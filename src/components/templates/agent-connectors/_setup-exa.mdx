Register your Exa API key with Scalekit so it can authenticate and proxy requests on behalf of your users. Exa uses API key authentication — no OAuth flow is required.

<Steps>
1. ### Generate an API key

   - Sign in to [dashboard.exa.ai/api-keys](https://dashboard.exa.ai/api-keys). In the left sidebar under **Management**, click **API Keys**.

   - Click **+ Create Key**, enter a name (e.g., `Agent Auth`), and confirm.

   - In the **Secret Key** column, click the eye icon to reveal the key and copy it.

      ![Exa dashboard API Keys page showing existing keys and the + Create Key button](@/assets/docs/agent-connectors/exa/create-api-key.png)

2. ### Create a connection in Scalekit

   - In [Scalekit dashboard](https://app.scalekit.com), go to **Agent Auth** → **Create Connection**. Find **Exa** and click **Create**.

      ![Scalekit connection configuration for Exa](@/assets/docs/agent-connectors/exa/add-credentials.png)

3. ### Add a connected account

   - Open the connection you just created and click the **Connected Accounts** tab, then click **Add account**.

   - Fill in the following fields:
     - **Your User's ID** — a unique identifier for this user in your system (e.g., `user_123`)
     - **API Key** — the key you copied in step 1

   - Click **Save**.

      ![Add connected account form for Exa in Scalekit dashboard](@/assets/docs/agent-connectors/exa/add-connected-account.png)
</Steps>

<Aside type="note">
**Credits and rate limits**: Each API key has a per-second QPS (queries per second) limit shown on the API Keys page — the default is 10 QPS. Every tool call consumes credits: search and crawl cost 1 credit per request plus additional credits per content item retrieved; research and websets are more expensive as they run multiple sub-queries internally. Monitor your usage at [dashboard.exa.ai](https://dashboard.exa.ai) → **Usage**.
</Aside>
