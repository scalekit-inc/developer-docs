Register your Scalekit environment with the Apollo connector so Scalekit handles the authentication flow and token lifecycle for you. The connection name you create will be used to identify and invoke the connection programmatically.

<Steps>
1. ### Set up auth redirects

    - In [Scalekit dashboard](https://app.scalekit.com), go to **Agent Auth** → **Create Connection**. Find **Apollo** and click **Create**. Click **Use your own credentials** and copy the redirect URI. It looks like `https://<SCALEKIT_ENVIRONMENT_URL>/sso/v1/oauth/<CONNECTION_ID>/callback`.

      ![](@/assets/docs/agent-connectors/apollo/use-own-credentials-redirect-uri.png)

    - Go to [Apollo Developer Settings](https://developer.apollo.io/) and sign in. Click **OAuth Applications** in the left sidebar, then click **Create OAuth Application**.

    - Fill in the application name and description. Under **Redirect URIs**, paste the redirect URI you copied from Scalekit.

      ![](@/assets/docs/agent-connectors/apollo/create-oauth-app.png)

    - Click **Create application**.

2. ### Get client credentials

    After creating the application, Apollo shows the **Client ID** and **Client Secret** on the application detail page. Copy both values — the Client Secret is only shown once.

    ![](@/assets/docs/agent-connectors/apollo/client-credentials.png)

3. ### Add credentials in Scalekit

    - In [Scalekit dashboard](https://app.scalekit.com), go to **Agent Auth** → **Connections** and open the connection you created.

    - Enter your credentials:
      - **Client ID** (from Apollo)
      - **Client Secret** (from Apollo)
      - **Permissions** — select the scopes your app needs (see [required scopes](#required-scopes) below)

    - Click **Save**.
</Steps>
