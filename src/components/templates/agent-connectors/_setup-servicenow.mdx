Register your Scalekit environment with ServiceNow so Scalekit handles the OAuth flow and token lifecycle for your users. Create an OAuth application registry in your ServiceNow instance, then add its Client ID and Client Secret to your Scalekit connection.

<Steps>
1. ### Copy the redirect URI from Scalekit

   - In the [Scalekit dashboard](https://app.scalekit.com), go to **AgentKit** > **Connections** > **Create Connection**. Find **ServiceNow** and click **Create**.

     <Aside type="tip" title="Testing vs. production">
       By default, Scalekit creates the connection using its own shared credentials. This works for testing. Before going to production, follow the steps below to use your own ServiceNow OAuth application.
     </Aside>

   - Click **Use your own credentials** and copy the redirect URI. It looks like `https://<SCALEKIT_ENVIRONMENT_URL>/sso/v1/oauth/<CONNECTION_ID>/callback`.

2. ### Open Application Registry in ServiceNow

   <Aside type="note" title="No ServiceNow instance yet?">
     Sign up for a free Personal Developer Instance at the [ServiceNow Developer Portal](https://developer.servicenow.com/) — click **Sign up**, then **Start Building** to provision an instance. It takes a few minutes to spin up.
   </Aside>

   - Log in to your ServiceNow instance.
   - Use the top navigation search to find **Application Registry** under **System OAuth**.

     ![ServiceNow navigation search showing System OAuth > Application Registry](@/assets/docs/agent-connectors/servicenow/navigate-application-registry.png)

   - The **Application Registries** list shows every OAuth client and provider already configured on your instance.

     ![Application Registries list showing existing OAuth clients and providers](@/assets/docs/agent-connectors/servicenow/application-registries-list.png)

3. ### Create a new OAuth application

   - Click **New**. ServiceNow asks **What kind of OAuth application?**

     ![OAuth application type picker with New Inbound Integration Experience highlighted](@/assets/docs/agent-connectors/servicenow/oauth-application-wizard.png)

   - Select **New Inbound Integration Experience**.

     <Aside type="caution" title="Skip the deprecated options">
       Options labeled **[Deprecated UI]** still work on some instances but ServiceNow no longer recommends them. Always choose **New Inbound Integration Experience** for new OAuth applications.
     </Aside>

   - Give the application a **Name** (for example, `Agent Auth`) and paste the redirect URI you copied in step 1 into the **Redirect URL** field.
   - Save the record.

4. ### Copy your Client ID and Client Secret

   - Open the application record you just created under **Application Registries**.
   - Copy the **Client ID** shown on the record.
   - Click the lock icon next to **Client Secret** to reveal and copy it.

     ![ServiceNow OAuth application record showing Client ID, Client Secret, and Redirect URL fields](@/assets/docs/agent-connectors/servicenow/oauth-client-credentials.png)

5. ### Add credentials in Scalekit

   - Return to the connection you created in Scalekit and enter:
     - **Client ID** — from your ServiceNow application registry
     - **Client Secret** — from your ServiceNow application registry
   - Click **Save**.
</Steps>
