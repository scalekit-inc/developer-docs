Circleback MCP uses OAuth 2.1 with Dynamic Client Registration (DCR) and PKCE — no client ID or secret is required. Scalekit registers the OAuth client automatically with the Circleback authorization server. The only step is creating a connection in Scalekit and authorizing a user account.

<Steps>
1. ### Create a connection in Scalekit

   - In the [Scalekit dashboard](https://app.scalekit.com), go to **AgentKit** → **Connections** → **Create Connection**.
   - Search for **Circleback MCP** and click **Create**.
   - Note the **Connection name** — use this as `connection_name` in your code (e.g., `circlebackmcp`).

   {/* TODO: add screenshot — alt: "Create connection dialog showing Circleback MCP", original src: @/assets/docs/agent-connectors/circlebackmcp/create-connection.png */}

2. ### Authorize your Circleback account

   Generate an authorization link and open it in a browser to complete the Circleback OAuth flow.

   The user is redirected to Circleback to sign in and grant access. Scalekit stores the token and automatically refreshes it for every tool call — no further configuration is needed.

   <Aside type="note" title="Circleback account required">
     The user must have an active Circleback account with meetings or transcripts to access. The OAuth flow uses PKCE and does not require storing a client secret.
   </Aside>
</Steps>
