Register your Scalekit environment with the Attio connector so Scalekit handles the authentication flow and token lifecycle for you. The connection name you create will be used to identify and invoke the connection programmatically. Then complete the configuration in your application as follows:

<Steps>
1. ### Set up auth redirects

    - In [Scalekit dashboard](https://app.scalekit.com), go to **Agent Auth** → **Create Connection**. Find **Attio** and click **Create**. Copy the redirect URI. It looks like `https://<SCALEKIT_ENVIRONMENT_URL>/sso/v1/oauth/<CONNECTION_ID>/callback`.

      ![Copy redirect URI from Scalekit dashboard](@/assets/docs/agent-connectors/attio/use-own-credentials-redirect-uri.png)

    - Log in to [Attio](https://app.attio.com) and go to **Settings** → **Developers** → **OAuth Apps**.

    - Open your OAuth app and paste the redirect URI into the **Redirect URIs** field. Click **Save changes**.

      ![Add redirect URI in Attio developer settings](@/assets/docs/agent-connectors/attio/add-redirect-uri.png)

2. ### Get client credentials

    - In your Attio OAuth app settings, copy the **Client ID** and **Client Secret**.

3. ### Add credentials in Scalekit

    - In [Scalekit dashboard](https://app.scalekit.com), go to **Agent Auth** → **Connections** and open the connection you created.

    - Enter your credentials:
      - Client ID (from above)
      - Client Secret (from above)
      - Permissions (scopes — see [Attio OAuth scopes reference](https://developers.attio.com/docs/authentication))

      ![Add credentials in Scalekit dashboard](@/assets/docs/agent-connectors/attio/add-credentials.png)
    - Click **Save**.
</Steps>
