Rize MCP uses OAuth 2.1 with Dynamic Client Registration (DCR) and PKCE — no client ID or secret is needed. Scalekit registers the OAuth client automatically and handles the full token lifecycle for every user who authorizes the connection.

<Steps>
1. ### Create a connection in Scalekit

   In the [Scalekit dashboard](https://app.scalekit.com), go to **AgentKit** > **Connections** > **Create Connection**. Search for **Rize MCP** and click **Create**.

   {/* TODO: add screenshot — alt: "Create Connection dialog with Rize MCP selected in Scalekit dashboard", original src: @/assets/docs/agent-connectors/rizemcp/create-connection.png */}

2. ### Authorize the connection

   Once the connection is created, click **Authorize**. Scalekit opens the Rize OAuth consent screen. Sign in with your Rize account and approve the requested permissions.

   {/* TODO: add screenshot — alt: "Rize OAuth consent screen during connection authorization", original src: @/assets/docs/agent-connectors/rizemcp/oauth-consent.png */}

   After authorization, Scalekit stores the token and the connection status changes to **Active**.

3. ### Verify the connection is active

   In **AgentKit** > **Connections**, confirm the Rize MCP connection shows status **Active**. You can now call `execute_tool` with any `rizemcp_*` tool name.

   {/* TODO: add screenshot — alt: "Rize MCP connection showing Active status in Scalekit dashboard", original src: @/assets/docs/agent-connectors/rizemcp/connection-active.png */}
</Steps>
