Scalekit ships pre-built tools for every connector in the catalog. Each tool returns a **structured, LLM-ready output** — no endpoint URLs to look up, no auth headers to inject, no response parsing. Your agent gets JSON it can reason over directly.

These tools cover 60+ connectors out of the box — Gmail, Slack, GitHub, Salesforce, Notion, Linear, HubSpot, and more. Browse the full list at the [Agent connectors catalog](/agentkit/connectors/) or in the dashboard under **Agent Auth > Connectors**.

## Before you start

Complete the [AgentKit quickstart](/agentkit/quickstart/) before continuing. This page assumes:

- The SDK is installed and a `ScalekitClient` is initialized
- A connector is configured with an active connected account for your user

<Steps>
1. ## Discover available tools

   Scalekit exposes four ways to find which tools are available. Pick the one that matches your need.

   ### Browse the connector catalog

   The [Agent connectors catalog](/agentkit/connectors/) and the dashboard's **Agent Auth > Connectors** page list every connector and the tools it ships with. Start here when designing your agent — it tells you which connectors cover your use case and what actions they expose.

   ### List every tool in your workspace

   `list_tools` / `listTools` returns all tools in your workspace, regardless of user authorization state. Use this for taking inventory or building admin UIs.

   <Tabs syncKey="tech-stack">
     <TabItem label="Python">
   ```python showLineNumbers=false frame="none"
   # Returns all tools registered in your Scalekit workspace
   response, _ = actions.tools.list_tools()
   for tool_name in response.tool_names:
       print(tool_name)
   ```
     </TabItem>
     <TabItem label="Node.js">
   ```ts showLineNumbers=false frame="none"
   // Returns all tools registered in your Scalekit workspace
   const response = await scalekit.tools.listTools();
   for (const toolName of response.toolNames) {
     console.log(toolName);
   }
   ```
     </TabItem>
   </Tabs>

   ### List tools available to a specific user

   `list_scoped_tools` / `listScopedTools` returns only the tools this user is authorized to call — filtered by their active connected accounts. **This is the list you pass to an LLM for tool calling.**

   <Tabs syncKey="tech-stack">
     <TabItem label="Python">
   ```python showLineNumbers=false frame="none"
   from google.protobuf.json_format import MessageToDict

   # Tools this user can call, filtered to Gmail
   scoped = actions.tools.list_scoped_tools(
       identifier="user_123",                       # same identifier used when creating the connected account
       filter={"connection_names": ["gmail"]},      # optional — omit to include all authorized connectors
   )
   scoped_response, _ = scoped
   for scoped_tool in scoped_response.tools:
       tool_definition = MessageToDict(scoped_tool.tool).get("definition", {})
       print(tool_definition.get("name"))
       print(tool_definition.get("description"))
       print(tool_definition.get("input_schema"))   # JSON Schema — pass directly to your LLM's tool-calling API
   ```
     </TabItem>
     <TabItem label="Node.js">
   ```ts showLineNumbers=false frame="none"
   // Tools this user can call, filtered to Gmail
   const { tools } = await scalekit.tools.listScopedTools('user_123', {
     filter: { connectionNames: ['gmail'] },        // optional — omit to include all authorized connectors
   });
   for (const tool of tools) {
     const definition = tool.tool.definition;
     console.log(definition.name);
     console.log(definition.description);
     console.log(definition.input_schema);          // JSON Schema — pass directly to your LLM's tool-calling API
   }
   ```
     </TabItem>
   </Tabs>

   ### Try tools in the playground

   The playground at **[app.scalekit.com](https://app.scalekit.com) > Agent Auth > Playground** lets you make live tool calls and inspect real response structures before writing any code. Select a connector, pick a tool, fill in the inputs, and run it — the response panel shows the exact JSON your agent will receive from `execute_tool` / `executeTool`.

   Use the playground to map out the fields you need before building your parsing logic.

2. ## Execute tools across connectors

   `execute_tool` / `executeTool` runs a named tool for a specific user. Scalekit injects the user's OAuth token, calls the provider's API, and returns structured output — your agent never touches raw credentials or API endpoints.

   The same pattern works for every connector — swap `tool_name` and `tool_input` for the target connector:

   | Connector | Tool name | Sample `tool_input` |
   |---|---|---|
   | `gmail` | `gmail_fetch_mails` | `{ "query": "is:unread", "max_results": 5 }` |
   | `slack` | `slack_send_message` | `{ "channel": "#general", "text": "Hello team" }` |
   | `github` | `github_create_issue` | `{ "repo": "acme/app", "title": "Bug report", "body": "Steps to reproduce…" }` |
   | `salesforce` | `salesforce_create_lead` | `{ "first_name": "Ada", "last_name": "Lovelace", "email": "ada@example.com" }` |

   <Aside type="note" title="Tool names are illustrative">
     Exact tool names and input schemas vary by connector. Call `list_scoped_tools` / `listScopedTools` or open the connector detail page in the dashboard to see the current schema for any tool.
   </Aside>

3. ## Wire the LLM tool-calling loop

   The complete agent loop: fetch scoped tools → hand them to the LLM → execute any tool call the LLM requests → feed the result back into the conversation.

   This example uses Anthropic's Messages API. The pattern is identical for OpenAI and Google tool calling — only the schema reshape and response format differ.

   <Tabs syncKey="tech-stack">
     <TabItem label="Python">
   ```python showLineNumbers=false frame="none"
   import os
   import anthropic
   from google.protobuf.json_format import MessageToDict

   client = anthropic.Anthropic()

   # 1. Fetch tools scoped to this user — only tools they are authorized to call
   scoped = actions.tools.list_scoped_tools(
       identifier="user_123",
       filter={"connection_names": ["gmail"]},
   )
   scoped_response, _ = scoped
   # Reshape into Anthropic's expected tool format
   llm_tools = [
       {
           "name": MessageToDict(t.tool).get("definition", {}).get("name"),
           "description": MessageToDict(t.tool).get("definition", {}).get("description"),
           "input_schema": MessageToDict(t.tool).get("definition", {}).get("input_schema", {}),
       }
       for t in scoped_response.tools
   ]

   # 2. Send the user's message to the LLM with the tool list attached
   messages = [{"role": "user", "content": "Summarize my last 5 unread emails"}]
   response = client.messages.create(
       model=os.getenv("LITELLM_MODEL", "claude-sonnet-4-6"),
       max_tokens=1024,
       tools=llm_tools,
       messages=messages,
   )

   # 3. Execute any tool_use blocks the LLM returned
   for block in response.content:
       if block.type == "tool_use":
           tool_result = actions.execute_tool(
               tool_name=block.name,
               identifier="user_123",
               tool_input=block.input,
           )
           # 4. Append the result back so the LLM can produce a final answer
           messages.append({"role": "assistant", "content": response.content})
           messages.append({
               "role": "user",
               "content": [{
                   "type": "tool_result",
                   "tool_use_id": block.id,
                   "content": str(tool_result.data),
               }],
           })
   ```
     </TabItem>
     <TabItem label="Node.js">
   ```ts showLineNumbers=false frame="none"
   import Anthropic from '@anthropic-ai/sdk';

   const anthropic = new Anthropic();

   // 1. Fetch tools scoped to this user — only tools they are authorized to call
   const { tools } = await scalekit.tools.listScopedTools('user_123', {
     filter: { connectionNames: ['gmail'] },
   });
   // Reshape into Anthropic's expected tool format
   const llmTools = tools.map((t) => ({
     name: t.tool.definition.name,
     description: t.tool.definition.description,
     input_schema: t.tool.definition.input_schema,
   }));

   // 2. Send the user's message to the LLM with the tool list attached
   const messages: Anthropic.MessageParam[] = [
     { role: 'user', content: 'Summarize my last 5 unread emails' },
   ];
   const response = await anthropic.messages.create({
     model: process.env.LITELLM_MODEL ?? 'claude-sonnet-4-6',
     max_tokens: 1024,
     tools: llmTools,
     messages,
   });

   // 3. Execute any tool_use blocks the LLM returned
   for (const block of response.content) {
     if (block.type === 'tool_use') {
       const toolResult = await scalekit.actions.executeTool({
         toolName: block.name,
         identifier: 'user_123',
         toolInput: block.input as Record<string, unknown>,
       });
       // 4. Append the result back so the LLM can produce a final answer
       messages.push({ role: 'assistant', content: response.content });
       messages.push({
         role: 'user',
         content: [{
           type: 'tool_result',
           tool_use_id: block.id,
           content: JSON.stringify(toolResult.data),
         }],
       });
     }
   }
   ```
     </TabItem>
   </Tabs>

4. ## Use a framework adapter

   For LangChain and Google ADK, Scalekit returns native tool objects in Python — no JSON Schema reshaping needed. In Node.js, use the Vercel AI SDK by reshaping `listScopedTools` output.

   <Tabs syncKey="framework-adapter">
     <TabItem label="LangChain">
   ```sh showLineNumbers=false frame="none"
   pip install scalekit-sdk-python langchain langchain-openai
   ```

   ```python showLineNumbers=false frame="none"
   from langchain_openai import ChatOpenAI
   from langchain.agents import create_agent
   import os

   # Native LangChain StructuredTool objects, scoped to this user
   tools = actions.langchain.get_tools(
       identifier="user_123",
       providers=["GMAIL"],      # filter to one or more providers
       page_size=100,
   )

   llm = ChatOpenAI(
       model=os.getenv("LITELLM_MODEL", "claude-sonnet-4-6"),
       openai_api_base=os.getenv("LITELLM_BASE_URL"),
       openai_api_key=os.getenv("LITELLM_API_KEY"),
   )
   agent = create_agent(
       model=llm,
       tools=tools,
       system_prompt="You are a helpful assistant. Use tools if needed.",
   )
   result = agent.invoke({
       "messages": [{"role": "user", "content": "fetch my last 5 unread emails and summarize them"}]
   })
   ```

   Full sample: [sample-langchain-agent](https://github.com/scalekit-inc/sample-langchain-agent).
     </TabItem>
     <TabItem label="Google ADK">
   ```sh showLineNumbers=false frame="none"
   pip install scalekit-sdk-python google-adk
   ```

   ```python showLineNumbers=false frame="none"
   import asyncio
   from google.adk.agents import Agent
   from google.adk.models.lite_llm import LiteLlm
   from google.adk.runners import Runner
   from google.adk.sessions import InMemorySessionService
   from google.genai import types

   # Native Google ADK tool objects, scoped to this user
   gmail_tools = actions.google.get_tools(
       identifier="user_123",
       providers=["GMAIL"],
       page_size=100,
   )

   gmail_agent = Agent(
       name="gmail_assistant",
       model=LiteLlm(model="claude-sonnet-4-6"),
       description="Gmail assistant that can read and manage emails",
       instruction="You are a helpful Gmail assistant. Focus on unread messages and provide clear summaries.",
       tools=gmail_tools,
   )

   async def run_agent(prompt: str):
       session_service = InMemorySessionService()
       session = await session_service.create_session(
           app_name="gmail_adk_example",
           user_id="user_123",
           session_id="gmail_adk_session_1",
       )
       runner = Runner(
           agent=gmail_agent,
           app_name="gmail_adk_example",
           session_service=session_service,
       )
       content = types.Content(role="user", parts=[types.Part(text=prompt)])
       async for event in runner.run_async(
           user_id="user_123",
           session_id=session.id,
           new_message=content,
       ):
           if event.is_final_response() and event.content and event.content.parts:
               print(event.content.parts[0].text)

   asyncio.run(run_agent("fetch my last 5 unread emails and summarize them"))
   ```

   Full sample: [google-adk-agent-example](https://github.com/scalekit-inc/google-adk-agent-example).
     </TabItem>
     <TabItem label="Node.js (Vercel AI SDK)">
   ```sh showLineNumbers=false frame="none"
   npm install ai @ai-sdk/openai-compatible
   ```

   ```ts showLineNumbers=false frame="none"
   import { generateText, jsonSchema, tool } from 'ai';
   import { createOpenAICompatible } from '@ai-sdk/openai-compatible';

   const { tools: scopedTools } = await scalekit.tools.listScopedTools('user_123', {
     filter: { connectionNames: ['gmail'] },
   });

   const tools = Object.fromEntries(
     scopedTools.map((t) => [
       t.tool.definition.name,
       tool({
         description: t.tool.definition.description,
         parameters: jsonSchema(t.tool.definition.input_schema ?? { type: 'object', properties: {} }),
         execute: async (args) => {
           const result = await scalekit.actions.executeTool({
             toolName: t.tool.definition.name,
             toolInput: args,
             identifier: 'user_123',
           });
           return result.data;
         },
       }),
     ]),
   );

   const litellm = createOpenAICompatible({
     baseURL: process.env.LITELLM_BASE_URL!,
     apiKey: process.env.LITELLM_API_KEY!,
     name: 'litellm',
   });

   const result = await generateText({
     model: litellm(process.env.LITELLM_MODEL ?? 'claude-sonnet-4-6'),
     tools,
     maxSteps: 5,
     prompt: 'fetch my last 5 unread emails and summarize them',
   });

   console.log(result.text);
   ```

## Inspect connected account details

To see a connected account's full auth details — access token, expiry, last used timestamp — call `get_connected_account` / `getConnectedAccount`. This method returns sensitive credentials; restrict it to server-side code.

<Tabs syncKey="tech-stack">
  <TabItem label="Python">
```python showLineNumbers=false frame="none"
# Returns sensitive credentials — server-side only
details = actions.get_connected_account(
    connection_name="gmail",
    identifier="user_123",
)
print(details.connected_account.token_expires_at)
print(details.connected_account.last_used_at)
```
  </TabItem>
  <TabItem label="Node.js">
```ts showLineNumbers=false frame="none"
// Returns sensitive credentials — server-side only
const details = await actions.getConnectedAccount({
  connectionName: 'gmail',
  identifier: 'user_123',
});
console.log(details.connectedAccount?.tokenExpiresAt);
console.log(details.connectedAccount?.lastUsedAt);
```
  </TabItem>
</Tabs>

See [connected account states](/agentkit/connected-accounts/#account-states) for the full status reference.

## Common AgentKit scenarios

<details>
<summary>Connected account stays in <code>PENDING_AUTH</code></summary>

Your connector exists, but the user has not completed OAuth consent yet.

What to do:

- Call `get_or_create_connected_account` / `getOrCreateConnectedAccount`.
- If status is not active, generate an authorization link with `get_authorization_link` / `getAuthorizationLink`.
- Redirect the user to the link and retry after consent.

Why this happens:

- The connector is configured, but no active token exists for that user identifier.

</details>

<details>
<summary>Tool calls fail with resource not found errors</summary>

Scalekit can return different exception classes depending on SDK path, but the root cause is usually one of these:

- The connector name does not exist in your workspace.
- The user identifier has no connected account for that connector.
- You are calling a tool name that is not available for that scoped user.

What to do:

- Verify connector setup under **Agent Auth > Connections**.
- Verify your identifier is the same one used during account creation.
- Call `list_scoped_tools` / `listScopedTools` and execute only returned tool names.

</details>

<details>
<summary>LiteLLM returns model access or budget errors</summary>

Your key might be restricted to specific models or monthly budget limits.

Typical errors:

- `key_model_access_denied`
- `budget_exceeded`

What to do:

- Set `LITELLM_MODEL` to a model your key can access.
- Handle budget and provider errors as warnings in verification runs.
- Keep tool discovery and direct `execute_tool` / `executeTool` checks separate from LLM-loop checks.

</details>

<details>
<summary>Connection names differ across environments</summary>

Connection names are workspace-specific. Do not hard-code assumptions from another environment.

For example:

- Gmail might be `gmail`
- GitHub might be `github-qkHFhMip`
- Linear might be `linear`

Recommended pattern:

- Configure connector names with environment variables such as `GMAIL_CONNECTION_NAME`, `GITHUB_CONNECTION_NAME`, and `LINEAR_CONNECTION_NAME`.
- Use those values in `list_scoped_tools` / `listScopedTools` filters and account APIs.

</details>

<details>
<summary>Use reference examples as your baseline</summary>

Use the [`agent-auth-examples` repository](https://github.com/scalekit-developers/agent-auth-examples) to confirm runtime behavior before updating production agent code or docs.

Start with the Python and JavaScript examples in that repository, then run the end-to-end and adapter flows that match this guide.

These scripts cover:

- End-to-end tool discovery and execution
- Connected account status handling
- Anthropic/LiteLLM tool-calling loop behavior
- LangChain, Google ADK, and Vercel AI SDK adapter flows

</details>
