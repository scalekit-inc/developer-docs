AgentKit lets your AI agents act on behalf of users across third-party applications — sending emails, reading calendars, creating tickets, querying databases, and more. Scalekit handles the authentication complexity (OAuth, API keys, RSA key pairs, service accounts, and others) and provides the tools your agent calls, so your code focuses on what the agent should do, not how to authenticate or which API endpoint to hit.

## Core concepts

**Connectors** are the third-party applications your agent can use — Gmail, Slack, Salesforce, Snowflake, GitHub, and many others. Each connector defines its authentication requirements and exposes tools your agent can call.

**Connections** are configurations you create once in the Scalekit Dashboard. A connection holds the credentials Scalekit needs to authenticate with a connector on your agent's behalf. One connection serves all your users.

**Connected accounts** are per-user instances of a connection. When a user completes authentication, Scalekit creates a connected account that holds their credentials and tracks their authorization state. Your agent uses a connected account to act on that specific user's behalf.

**Tools** are pre-built, connector-specific actions your agent can call — for example, `gmail_fetch_emails` or `salesforce_create_record`. Scalekit provides the tool schemas and handles the authenticated API call. Your agent passes inputs; Scalekit injects the user's credentials and returns structured output.

## How they fit together

```d2
direction: right

Dev: "You (developer)"
Users: "Your users"
Agent: "Your agent"

Scalekit: {
  Conn: "Connection\n(credentials + config)"
  CA: "Connected accounts\n(per-user auth state)"
  Tools: "Tools\n(pre-built + proxied)"
}

Provider: "Third-party app\n(Gmail, Slack, Snowflake…)"

Dev -> Scalekit.Conn: "Configure once"
Users -> Scalekit.CA: "Authenticate"
Agent -> Scalekit.Tools: "Call tools"
Scalekit.Tools -> Provider: "Authenticated API call"
```

You configure connections once. Your users authenticate to create connected accounts. Your agent calls tools — Scalekit handles the rest.

## What Scalekit manages for you

**Authentication**
- Credential management across OAuth, API keys, RSA key pairs, service accounts, and more
- Secure credential storage and automatic token refresh
- Per-user authorization state tracking

**Tool execution**
- Pre-built tool schemas for every connector — pass directly to any LLM API
- Authenticated API proxying — your agent never handles raw tokens or constructs API calls
- Native adapters for LangChain, Google ADK, and MCP-compatible frameworks
