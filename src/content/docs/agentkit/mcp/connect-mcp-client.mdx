Before connecting your agent, each user must have authorized access to the external accounts (Gmail, Google Calendar, and so on) that your Virtual MCP Server uses. Once all connections are active, you mint a session token and pass it to your agent alongside the server URL.

This pattern works with any programmatic agent framework: Claude Managed Agents, OpenAI Agents, Hermes Agents, or any MCP-compatible agent that accepts a Streamable HTTP endpoint with bearer auth.

<Steps>

1. ## Check that connections are active

   Each user connects their external accounts once. All future agent sessions for that user reuse those connections. Before each agent run, verify that all required connections are `"ACTIVE"`.

   ```python
   # Retrieve config_id by listing Virtual MCP Servers filtered by name to use below
   accounts_response = scalekit_client.actions.mcp.list_mcp_connected_accounts(
       config_id=config_id,
       identifier="user_123",     # your app's unique user ID
   )
   for account in accounts_response.connected_accounts:
       if account.connected_account_status != "ACTIVE":
           print(f"{account.connection_name} is not active — prompt user to authorize")
   ```

   If any connection is not `"ACTIVE"`, prompt the user to authorize before proceeding. See [Authorize user connections](/agentkit/tools/authorize/).

2. ## Mint a session token

   Mint a fresh session token before each agent run. Never reuse tokens across sessions.

   ```python
   from datetime import timedelta

   token_response = scalekit_client.actions.mcp.create_session_token(
       mcp_config_id=config_id,
       identifier="user_123",     # your app's unique user ID
       expiry=timedelta(hours=1),
   )
   token = token_response.token
   ```

3. ## Pass the token as bearer auth

   Pass the Virtual MCP Server URL and the session token to your agent. The agent connects using Streamable HTTP transport with bearer auth and discovers available tools automatically.

   ```python
   # Pass to your agent framework (Claude Managed Agents, OpenAI Agents SDK, Hermes, or any MCP-compatible agent)
   {
       "url": mcp_server_url,        # from vmcp_response.config.mcp_server_url
       "headers": {"Authorization": f"Bearer {token}"},
   }
   ```

   Refer to your agent framework's documentation for how to register an MCP server URL and pass custom headers.

</Steps>

## Best practices

**Always mint a fresh token before each session.** Session tokens are scoped to a user and an expiry window. Reusing a token from a previous session risks it expiring mid-run. Create a new token at the start of every agent invocation.

**Set expiry with a buffer.** Token expiry should exceed the expected agent run duration. If your agent typically finishes in 20 minutes, set a 30-minute expiry. This avoids mid-run token expiration without leaving long-lived tokens exposed.

**Update Virtual MCP Servers only when idle.** Updating a Virtual MCP Server while agent sessions are running can cause tools to become unavailable mid-execution. Wait for all active sessions to complete before updating. For significant changes, create a new Virtual MCP Server and swap the URL in your agent definition instead of updating in place.

**Verify connections before each run.** Call `list_mcp_connected_accounts` before minting a token to confirm all connections are still `ACTIVE`. OAuth tokens for connected accounts can expire or be revoked. If a connection is inactive, surface an authorization link to the user before starting the agent.
