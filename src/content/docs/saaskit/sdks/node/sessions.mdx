Use the `session` client to inspect and revoke user sessions.

### `getSession`

<div class="sdk-method-section">
  <CB.ClassBrowser name="Sessions" type="client" language="typescript" source="https://github.com/scalekit-inc/scalekit-sdk-node/blob/main/src/session.ts">
    <CB.ClassMethod name="getSession" open={true}>
      <CB.ClassModifier type="async" slot="modifiers" />

      Retrieves comprehensive metadata and status for a specific user session.

      <CB.ClassMethodInput name="sessionId" type="string">
        The session identifier to retrieve (format: "ses_...")
      </CB.ClassMethodInput>
      <CB.ClassMethodOutput type="SessionDetails">
        Session details containing: - sessionId: Unique session identifier - userId: User who owns this session - organizationId: Organization context for the sessio…
      </CB.ClassMethodOutput>

```typescript wrap showLineNumbers=false
const session = await scalekit.session.getSession('ses_123456');

console.log('Status:', session.status);
console.log('User:', session.userId);
console.log('Device:', session.deviceInfo);
console.log('Location:', session.geoLocation);
```

      <CB.ClassMethodInput name="userId" type="string">
        userId parameter
      </CB.ClassMethodInput>
      <CB.ClassMethodInput name="options" type="object" hint="pageSize, pageToken, filter, status, startTime, endTime">
        Optional.
      </CB.ClassMethodInput>
      <CB.ClassMethodOutput type="UserSessionDetails">
        UserSessionDetails
      </CB.ClassMethodOutput>

```typescript wrap showLineNumbers=false
const response = await scalekit.session.getUserSessions('usr_123456', {
  filter: { status: ['active'] },
  pageSize: 20
});
```
