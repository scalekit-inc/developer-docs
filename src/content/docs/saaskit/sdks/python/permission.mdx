Use `scalekit_client.permissions` to define fine-grained permissions that roles can grant.

Model permissions for resources in your app, then attach them to roles.

### list_permissions
<div class="sdk-method-section">
  <CB.ClassBrowser name="Permission" type="client" language="python" source="https://github.com/scalekit-inc/scalekit-sdk-python/blob/main/scalekit/permissions.py">
    <CB.ClassMethod name="list_permissions" open={true}>
      <CB.ClassModifier type="async" slot="modifiers" />

      Method to list all permissions

      <CB.ClassMethodInput name="page_token" type="Optional[str]">
        Optional. Token for pagination
      </CB.ClassMethodInput>
      <CB.ClassMethodInput name="page_size" type="Optional[int]">
        Optional. Number of permissions per page
      </CB.ClassMethodInput>
      <CB.ClassMethodOutput type="ListPermissionsResponse">
        List Permissions Response
      </CB.ClassMethodOutput>

```python wrap showLineNumbers=false
response = scalekit_client.permissions.list_permissions(
    page_size=50
)

    print(f"Permission: {permission.name}")
```
