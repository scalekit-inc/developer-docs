        1. ## Set up developer environment

        2. ## Starting prompt

           Now ask your agent to implement Scalekit auth in natural language. Start with one of these prompts and adapt it to your app:

           <Tabs syncKey="starter-prompt">
             <TabItem label="HubSpot + Gmail">
               ```text title="Expand my agent with HubSpot and Gmail" showLineNumbers=false
               Expand my current agent code so it can connect to HubSpot and Gmail with Scalekit Agent Auth.

               Start by reading the Agent Auth quickstart, the HubSpot connector guide, and the Gmail examples in the docs. Then guide me step by step to:
               1. Configure the required Scalekit connections.
               2. Create or reuse the connected account for my user.
               3. Trigger the authorization flow if the user is not connected.
               4. Fetch the latest 5 emails from Gmail.
               5. Keep the implementation ready for HubSpot actions next.

               Use production-safe patterns, explain security implications, and update my code instead of giving me a high-level summary.
               ```
             </TabItem>
             <TabItem label="Access + refresh tokens">
               ```text title="Retrieve tokens for direct API calls" showLineNumbers=false
               Guide me step by step to implement Scalekit so I can securely retrieve a user's access token and refresh token when I need to call a provider API directly.

               Read the connected accounts and token management docs first. Then:
               1. Add the required Scalekit setup to my app.
               2. Create or load the connected account for my user.
               3. Trigger authorization when the account is not active.
               4. Show me how to read `accessToken` and `refreshToken` from the connected account details.
               5. Explain when I should call the provider directly versus using Scalekit's proxy or built-in tools.

               Use secure defaults, never expose tokens in logs, and make the final code ready for real API calls.
               ```
             </TabItem>
             <TabItem label="LangChain + Calendar">
               ```text title="Use LangChain with Google Calendar" showLineNumbers=false
               Use LangChain as the primary framework and help me build an agent with Scalekit that connects to Google Calendar.

               Start from the LangChain framework guide and the Google Calendar examples. Then work step by step to:
               1. Configure the `googlecalendar` connection in Scalekit.
               2. Create or reuse the connected account for my user.
               3. Send the user through authorization if needed.
               4. Retrieve a valid access token from Scalekit or use the built-in tool if that is the better pattern.
               5. Call Google Calendar and return upcoming events.

               Write the implementation code, explain any required scopes, and keep the flow ready for follow-up actions such as creating calendar events.
               ```
             </TabItem>
           </Tabs>

        </Steps>
