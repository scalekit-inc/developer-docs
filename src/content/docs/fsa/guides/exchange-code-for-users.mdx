When users complete verifying their identity based on their selected login method, Scalekit redirects to your registered callback URL with either an authorization code (success) or error parameters (failure).

Your endpoint needs to exchange the authorization code for user profile data  and manage the next authentication steps.

The `authResult` object returned contains:

- `user` - Common user details with email, name, and verification status
- `idToken` - JWT containing verified full user identity claims
- `accessToken` - Short-lived token for API authorization
- `refreshToken` - Long-lived token to obtain new access tokens

<AuthResultTabsSection />

The authentication response contains user details encoded in JWT tokens. These tokens can be decoded to retrieve comprehensive user information. Additionally, Scalekit automatically stores user details in your environment, enabling easy access at any time.
