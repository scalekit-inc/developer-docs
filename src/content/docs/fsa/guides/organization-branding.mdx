When a user from Acme Corp signs in, they see Acme Corp's logo — not your platform's logo. Organization branding replaces your workspace branding with the individual customer's logo and name across three surfaces: the hosted login page, the admin portal, and the end-user app.

## Where it appears

| Surface | What changes |
|---|---|
| **Hosted login page** (`/a`) | Logo and "Login to \{org name\}" heading on the login page only. The signup page always shows your workspace logo. |
| **Admin portal** (`/portals`) | Logo in the portal header. The title remains "Admin Portal". |
| **End-user app** (`/ui`) | Logo in the top navigation header. |

## Enable the feature

Organization branding is controlled by a feature flag in **Customization > Branding > Organization branding** in your Scalekit dashboard. Toggle it on to activate it for your environment.

Once enabled, any organization that has a `logo_url` set will display its own logo on the surfaces above. Organizations without a `logo_url` continue to show your workspace logo.

## Set an organization's logo

Set `logo_url` when creating or updating an organization. The URL must be publicly accessible — Scalekit does not host or proxy logo images.

### When creating an organization

<Tabs syncKey="tech-stack">
<TabItem label="Node.js" value="nodejs">
```javascript title="Create organization with logo"
const organization = await scalekit.organization.createOrganization(
  'Acme Corporation',
  {
    externalId: 'acme-corp-123',
    logoUrl: 'https://cdn.acmecorp.com/logo.png',
  }
);

```
</TabItem>
<TabItem label="Python" value="python">
```python title="Create organization with logo"
from scalekit.v1.organizations.organizations_pb2 import CreateOrganization

organization = scalekit_client.organization.create_organization(
    CreateOrganization(
        display_name='Acme Corporation',
        external_id='acme-corp-123',
        logo_url='https://cdn.acmecorp.com/logo.png',
    )
)

print(f'Organization created: {organization.id}')
```
</TabItem>
<TabItem label="Go" value="go">
```go title="Create organization with logo"
logoURL := "https://cdn.acmecorp.com/logo.png"

organization, err := scalekitClient.Organization.CreateOrganization(
    ctx,
    "Acme Corporation",
    scalekit.CreateOrganizationOptions{
        ExternalId: "acme-corp-123",
        LogoUrl:    &logoURL,
    },
)
if err != nil {
    log.Fatal(err)
}

fmt.Printf("Organization created: %s\n", organization.ID)
```
</TabItem>
<TabItem label="Java" value="java">
```java title="Create organization with logo"
CreateOrganization createOrg = CreateOrganization.newBuilder()
    .setDisplayName("Acme Corporation")
    .setExternalId("acme-corp-123")
    .setLogoUrl("https://cdn.acmecorp.com/logo.png")
    .build();

Organization organization = scalekitClient.organizations().create(createOrg);
System.out.println("Organization created: " + organization.getId());
```
</TabItem>

### When updating an existing organization

<Tabs syncKey="tech-stack">
<TabItem label="Node.js" value="nodejs">
```javascript title="Update org logo"
const updatedOrg = await scalekit.organization.updateOrganization(
  'org_12345',
  { logoUrl: 'https://cdn.acmecorp.com/logo.png' }
);
```
</TabItem>
<TabItem label="Python" value="python">
```python title="Update org logo"
from scalekit.v1.organizations.organizations_pb2 import UpdateOrganization

updated_org = scalekit_client.organization.update_organization(
    organization_id='org_12345',
    organization=UpdateOrganization(
        logo_url='https://cdn.acmecorp.com/logo.png'
    )
)
```
</TabItem>
<TabItem label="Go" value="go">
```go title="Update org logo"
logoURL := "https://cdn.acmecorp.com/logo.png"

updatedOrg, err := scalekitClient.Organization.UpdateOrganization(
    ctx,
    "org_12345",
    &scalekit.UpdateOrganization{
        LogoUrl: &logoURL,
    },
)
if err != nil {
    log.Fatal(err)
}
```
</TabItem>
<TabItem label="Java" value="java">
```java title="Update org logo"
UpdateOrganization update = UpdateOrganization.newBuilder()
    .setLogoUrl("https://cdn.acmecorp.com/logo.png")
    .build();

Organization updatedOrg = scalekitClient.organizations()
    .updateById("org_12345", update);
```
</TabItem>
</Tabs>

To remove a logo, set `logo_url` to an empty string. The surface falls back to your workspace logo.

## Fallback behavior

If organization logo branding is enabled but an organization has no `logo_url`, the hosted surfaces fall back to your workspace logo. There is no broken image state — the feature degrades gracefully per organization.

<Aside type="note">
Organization branding requires the feature flag to be enabled at the environment level. Enabling it for individual organizations without the environment-level flag has no effect.
</Aside>

## Relationship to workspace branding

Workspace branding (configured in **Customization > Branding**) sets your platform's default logo, colors, and layout. Organization branding overrides the logo and display name only — colors, border radius, and layout remain unchanged from your workspace settings.
