Scalekit's quickstart shows how new tenants are created during sign-up. This guide extends that flow by covering every organization creation path after you finish the [installation guide](/authenticate/installation/), the [FSA quickstart](/fsa/quickstart/), and [manage user sessions](/fsa/guides/manage-session/).

You can create organizations in three ways:

- Use the Scalekit dashboard when operators need to create or seed organizations manually.
- Let hosted signup create an organization the first time a new customer signs up.
- Call the SDK or REST APIs from your application to create organizations programmatically.

Sign-up restrictions decide who is allowed to start new organizations. Complete the [signup restrictions guide](/fsa/guides/signup-restrictions/) before rolling out self-service creation.

## Control who can create organizations

Sign-up restrictions block disposable addresses and enforce invitation-only flows. Configure these policies in the [signup restrictions guide](/fsa/guides/signup-restrictions/) before enabling self-service signup.

### Understand restriction options

Scalekit enforces three levels of control:

- **Public domains** allow any user to create a new organization during hosted signup.
- **Allowed corporate domains** restrict signup to trusted business domains you specify.
- **Invitations only** block self-service signup and require admins to invite new members.

These settings decide whether Scalekit automatically creates an organization when a new email arrives. Tightened rules stop sprawl and make sure every organization starts with a verified domain.

<Aside type="tip">
Lock down organization creation for admin-managed tenants. This limits lateral movement if credentials leak and keeps invoices tied to verified companies.
</Aside>

## Create organizations in the Scalekit dashboard

Use the dashboard when operators or support teams need to seed workspaces or help customers onboard.

<Steps>

 1. ## Open the organization workspace

    Sign in to the Scalekit dashboard and navigate to **Dashboard** > **Organizations**. Choose **Create organization**.

 2. ## Enter organization details

    Provide the organization name, external identifier, and any tags you use for reporting or billing. You can edit these fields later in [organization settings](/fsa/guides/organization-settings/).

 3. ## Configure optional features

    Enable optional integrations that your customer requested:

    - **SSO**: Follow the [enterprise SSO guide](/fsa/guides/enterprise-sso/) to link an identity provider before inviting users.
    - **SCIM provisioning**: Prepare automated user sync with the [just-in-time provisioning guide](/fsa/guides/just-in-time-provisioning/).
    - **Metadata**: Store billing plan or CRM IDs in the metadata fields. Map them later through [organization settings](/fsa/guides/organization-settings/).

 4. ## Invite administrators

    Add the first administrators or seed users. Invitations respect your signup restrictions and enforce trusted domains.

</Steps>

<Aside type="tip">
Test organization creation in a sandbox environment before rolling out to production. Seed test tenants, validate SSO, and remove them once workflows pass QA.
</Aside>

## Automate organization creation with the SDK <Badge text="API" variant="tip" />

Call the Scalekit SDK or REST API to create organizations during onboarding flows or migrations. Review the [Organizations API reference](https://docs.scalekit.com/apis/#tag/organizations) for a full parameter list.

<Tabs syncKey="tech-stack">
  <TabItem value="node" label="Node.js">

```javascript title="Create and manage organizations" wrap
// ⚠️ SECURITY: Load credentials from environment variables or secret manager
import { Scalekit } from '@scalekit-sdk/node'

const client = new Scalekit({
  environmentUrl: process.env.SCALEKIT_ENVIRONMENT_URL,
  clientId: process.env.SCALEKIT_CLIENT_ID,
  clientSecret: process.env.SCALEKIT_CLIENT_SECRET
})

// Use case: Provision a workspace after a sales-assisted onboarding
const { organization } = await client.organization.createOrganization({
  name: 'Orion Analytics',
  externalId: 'crm-9342',
  metadata: { billingTier: 'enterprise' }
})

// Use case: Update metadata when billing status changes
await client.organization.updateOrganization(organization.id, {
  metadata: { billingTier: 'enterprise', seatsPurchased: 120 }
})

// Use case: Sync organization profile to downstream systems
const { organization: fetched } = await client.organization.getOrganization(organization.id)
console.info('Fetched organization', fetched)

// ⚠️ SECURITY: Load credentials from environment variables or secret manager
package main

import (
    "context"
    "fmt"
    "log"
    "os"

    scalekit "github.com/scalekit/scalekit-go"
)

func main() {
    ctx := context.Background()
    client, err := scalekit.NewClient(ctx, scalekit.Config{
        EnvironmentURL: os.Getenv("SCALEKIT_ENVIRONMENT_URL"),
        ClientID:       os.Getenv("SCALEKIT_CLIENT_ID"),
        ClientSecret:   os.Getenv("SCALEKIT_CLIENT_SECRET"),
    })
    if err != nil {
        log.Fatalf("failed to create client: %v", err)
    }

    // Use case: Provision a workspace after a sales-assisted onboarding
    created, err := client.Organization.Create(ctx, scalekit.CreateOrganizationRequest{
        Name:       "Orion Analytics",
        ExternalID: scalekit.String("crm-9342"),
        Metadata: map[string]string{
            "billingTier":    "enterprise",
            "seatsPurchased": "100",
        },
    })
    if err != nil {
        log.Fatalf("create organization: %v", err)
    }

    // Use case: Update metadata when billing status changes
    if _, err = client.Organization.Update(ctx, created.Organization.Id, scalekit.UpdateOrganizationRequest{
        Metadata: map[string]string{
            "billingTier":    "enterprise",
            "seatsPurchased": "120",
        },
    }); err != nil {
        log.Fatalf("update organization: %v", err)
    }

    // Use case: Sync organization profile to downstream systems
    fetched, err := client.Organization.Get(ctx, created.Organization.Id)
    if err != nil {
        log.Fatalf("get organization: %v", err)
    }
    fmt.Printf("Fetched organization: %+v\n", fetched.Organization)
}
```

import com.scalekit.ScalekitClient;
import com.scalekit.models.Organization;
import com.scalekit.requests.CreateOrganizationRequest;
import com.scalekit.requests.UpdateOrganizationRequest;

public class OrganizationExample {
  public static void main(String[] args) {
    ScalekitClient client = ScalekitClient.builder()
        .environmentUrl(System.getenv("SCALEKIT_ENVIRONMENT_URL"))
        .clientId(System.getenv("SCALEKIT_CLIENT_ID"))
        .clientSecret(System.getenv("SCALEKIT_CLIENT_SECRET"))
        .build();

    // Use case: Provision a workspace after a sales-assisted onboarding
    Organization organization = client.organization().create(
        CreateOrganizationRequest.builder()
            .name("Orion Analytics")
            .externalId("crm-9342")
            .putMetadata("billingTier", "enterprise")
            .build()
    ).organization();

    // Use case: Update metadata when billing status changes
    client.organization().update(
        organization.id(),
        UpdateOrganizationRequest.builder()
            .putMetadata("billingTier", "enterprise")
            .putMetadata("seatsPurchased", "120")
            .build()
    );

    // Use case: Sync organization profile to downstream systems
    Organization fetched = client.organization().get(organization.id()).organization();
    System.out.printf("Fetched organization: %s%n", fetched);
  }
}

## Let hosted signup create organizations automatically

The [FSA quickstart](/fsa/quickstart/) explains how hosted signup creates an organization for brand-new customers. When the first user signs up and your sign-up restrictions allow their email domain, Scalekit creates an organization and assigns the user as an administrator.

Direct users to the hosted signup page using the same authorization URL pattern from the quickstart:

## Allow authenticated users to start additional organizations

Some customers expand into multiple workspaces. Provide clear entry points such as a **New workspace** button in your dashboard or a guided onboarding flow.

Role-based access control (RBAC) decides who can launch additional organizations. Grant the capability to admins only and record every action for auditability. Consider adding rate limits to prevent automated abuse.

<Tabs syncKey="tech-stack">
  <TabItem value="node" label="Node.js">

```javascript title="Create new workspace for authenticated user" wrap
export const createNewWorkspace = async (req, res) => {
  const userId = req.session.userId

  // Use case: Allow admins to spin up regional workspaces
  const organization = await scalekit.organization.createOrganization(
    `${req.body.workspaceName} Workspace`,
    {
      externalId: `workspace-${Date.now()}`,
      metadata: {
        parentOrganizationId: req.session.organizationId,
        createdBy: userId,
      }
    }
  )

  // Add the creator as an admin of the new workspace
  await scalekit.membership.createMembership({
    organizationId: organization.id,
    userId,
    role: 'admin',
    sendInvitationEmail: false,
  })

  audit.log('organization.created', { organizationId: organization.id, userId })
  res.json({ organization })
}
```

func createNewWorkspace(ctx context.Context, req CreateWorkspaceRequest) (*Organization, error) {
    userID := req.Session.UserID

    // Use case: Allow admins to spin up regional workspaces
    created, err := scalekit.Organization.Create(ctx, scalekit.CreateOrganizationRequest{
        Name: fmt.Sprintf("%s Workspace", req.Body.WorkspaceName),
        ExternalID: scalekit.String(fmt.Sprintf("workspace-%d", time.Now().Unix())),
        Metadata: map[string]string{
            "parentOrganizationId": req.Session.OrganizationID,
            "createdBy":           userID,
        },
    })
    if err != nil {
        return nil, err
    }

    // Add the creator as an admin of the new workspace
    if _, err = scalekit.Membership.Create(ctx, scalekit.CreateMembershipRequest{
        OrganizationID: created.Organization.Id,
        UserID:         userID,
        Role:           scalekit.String("admin"),
        SendInvitationEmail: scalekit.Bool(false),
    }); err != nil {
        return nil, err
    }

    audit.Log("organization.created", map[string]string{
        "organizationId": created.Organization.Id,
        "userId":         userID,
    })

    return &created.Organization, nil
}
