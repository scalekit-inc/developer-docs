This page provides comprehensive documentation for all available tools in Agent Connect. Each tool includes detailed parameter specifications, response formats, and usage examples.

## Tool reference format

Each tool definition includes:

- **Name and description**: What the tool does
- **Provider**: Which third-party service it connects to
- **Category**: Functional grouping (communication, calendar, etc.)
- **Input schema**: Required and optional parameters
- **Output schema**: Response format and fields
- **Examples**: Common usage patterns
- **Error codes**: Specific error conditions

## Gmail tools

### send_email

Send an email message to one or more recipients.

**Parameters:**
- `to` (required): Array of recipient email addresses
- `subject` (required): Email subject line
- `body` (required): Plain text email body
- `html_body` (optional): HTML formatted email body
- `cc` (optional): Array of CC email addresses
- `bcc` (optional): Array of BCC email addresses
- `attachments` (optional): Array of file attachments
- `reply_to` (optional): Reply-to email address

**Response:**
- `message_id`: Unique identifier for the sent message
- `status`: Status of the email ('sent', 'queued', 'failed')
- `sent_at`: Timestamp when email was sent

**Example:**
```javascript
await agentConnect.tools.execute({
  connected_account_id: 'gmail_account',
  tool: 'send_email',
  parameters: {
    to: ['recipient@example.com'],
    subject: 'Important Update',
    body: 'Please review the attached document.',
    html_body: '<p>Please review the <strong>attached</strong> document.</p>',
    cc: ['manager@example.com'],
    attachments: [
      {
        filename: 'document.pdf',
        content: 'base64encodedcontent',
        mime_type: 'application/pdf'
      }
    ]
  }
});
```

### fetch_emails

Retrieve email messages from the user's mailbox.

**Parameters:**
- `query` (optional): Gmail search query (default: all emails)
- `max_results` (optional): Maximum number of emails to return (default: 10)
- `page_token` (optional): Token for pagination
- `include_attachments` (optional): Include attachment data (default: false)
- `format` (optional): Response format ('minimal', 'full', 'metadata')

**Response:**
- `emails`: Array of email objects
- `next_page_token`: Token for next page of results
- `total_count`: Total number of matching emails

**Example:**
```javascript
await agentConnect.tools.execute({
  connected_account_id: 'gmail_account',
  tool: 'fetch_emails',
  parameters: {
    query: 'is:unread from:support@example.com',
    max_results: 25,
    format: 'full'
  }
});
```

### create_draft

Create an email draft without sending it.

**Parameters:**
- `to` (required): Array of recipient email addresses
- `subject` (required): Email subject line
- `body` (required): Plain text email body
- `html_body` (optional): HTML formatted email body
- `cc` (optional): Array of CC email addresses
- `bcc` (optional): Array of BCC email addresses

**Response:**
- `draft_id`: Unique identifier for the draft
- `message_id`: Associated message ID
- `created_at`: Timestamp when draft was created

### reply_to_email

Reply to an existing email message.

**Parameters:**
- `message_id` (required): ID of the message to reply to
- `body` (required): Reply message body
- `html_body` (optional): HTML formatted reply body
- `reply_all` (optional): Reply to all recipients (default: false)

**Response:**
- `message_id`: Unique identifier for the reply
- `status`: Status of the reply
- `sent_at`: Timestamp when reply was sent

## Google Calendar tools

### create_event

Create a new calendar event.

**Parameters:**
- `title` (required): Event title/summary
- `start_time` (required): Event start time (ISO 8601)
- `end_time` (required): Event end time (ISO 8601)
- `description` (optional): Event description
- `location` (optional): Event location
- `attendees` (optional): Array of attendee email addresses
- `timezone` (optional): Event timezone
- `recurrence` (optional): Recurrence rules
- `reminders` (optional): Reminder settings

**Response:**
- `event_id`: Unique identifier for the event
- `html_link`: Link to view event in Google Calendar
- `created_at`: Timestamp when event was created

**Example:**
```javascript
await agentConnect.tools.execute({
  connected_account_id: 'calendar_account',
  tool: 'create_event',
  parameters: {
    title: 'Project Planning Meeting',
    description: 'Monthly project planning session',
    start_time: '2024-01-15T14:00:00Z',
    end_time: '2024-01-15T15:30:00Z',
    location: 'Conference Room A',
    attendees: ['team@example.com', 'manager@example.com'],
    recurrence: {
      frequency: 'monthly',
      interval: 1
    },
    reminders: [
      { method: 'email', minutes: 60 },
      { method: 'popup', minutes: 10 }
    ]
  }
});
```

### fetch_events

Retrieve calendar events within a specified time range.

**Parameters:**
- `start_date` (required): Start date for event search (ISO 8601)
- `end_date` (required): End date for event search (ISO 8601)
- `max_results` (optional): Maximum number of events to return
- `calendar_id` (optional): Specific calendar ID (default: primary)
- `single_events` (optional): Expand recurring events (default: true)

**Response:**
- `events`: Array of calendar event objects
- `next_page_token`: Token for pagination
- `total_count`: Total number of events in range

### update_event

Update an existing calendar event.

**Parameters:**
- `event_id` (required): ID of the event to update
- `title` (optional): New event title
- `start_time` (optional): New start time
- `end_time` (optional): New end time
- `description` (optional): New description
- `location` (optional): New location
- `attendees` (optional): New attendee list

**Response:**
- `event_id`: Updated event ID
- `updated_at`: Timestamp when event was updated

### find_available_slots

Find available time slots in a calendar.

**Parameters:**
- `start_date` (required): Start date for availability search
- `end_date` (required): End date for availability search
- `duration` (required): Duration of slots in minutes
- `attendees` (optional): Check availability for specific attendees
- `working_hours` (optional): Define working hours constraints

**Response:**
- `available_slots`: Array of available time slots
- `busy_periods`: Array of busy time periods

## Slack tools

### send_message

Send a message to a Slack channel or user.

**Parameters:**
- `channel` (required): Channel name, ID, or user ID
- `text` (required): Message content
- `attachments` (optional): Rich message attachments
- `blocks` (optional): Block Kit UI components
- `thread_ts` (optional): Send as thread reply
- `unfurl_links` (optional): Unfurl links in message

**Response:**
- `message_ts`: Timestamp of the sent message
- `channel`: Channel where message was sent
- `permalink`: Permanent link to the message

**Example:**
```javascript
await agentConnect.tools.execute({
  connected_account_id: 'slack_account',
  tool: 'send_message',
  parameters: {
    channel: '#general',
    text: 'Deployment completed successfully!',
    attachments: [
      {
        color: 'good',
        title: 'Production Deployment',
        text: 'Version 2.1.0 has been deployed to production',
        fields: [
          { title: 'Version', value: '2.1.0', short: true },
          { title: 'Environment', value: 'Production', short: true }
        ],
        ts: Math.floor(Date.now() / 1000)
      }
    ]
  }
});
```

### fetch_messages

Retrieve messages from a Slack channel.

**Parameters:**
- `channel` (required): Channel name or ID
- `limit` (optional): Number of messages to retrieve (default: 100)
- `oldest` (optional): Start of time range (timestamp)
- `latest` (optional): End of time range (timestamp)
- `inclusive` (optional): Include messages at boundaries

**Response:**
- `messages`: Array of message objects
- `has_more`: Whether more messages are available
- `response_metadata`: Metadata for pagination

### create_channel

Create a new Slack channel.

**Parameters:**
- `name` (required): Channel name (without #)
- `is_private` (optional): Create private channel (default: false)
- `topic` (optional): Channel topic
- `purpose` (optional): Channel purpose

**Response:**
- `channel_id`: ID of the created channel
- `channel_name`: Name of the created channel
- `created_at`: Timestamp when channel was created

### upload_file

Upload a file to Slack.

**Parameters:**
- `file_content` (required): File content (base64 encoded)
- `filename` (required): Name of the file
- `channels` (optional): Channels to share file with
- `title` (optional): File title
- `initial_comment` (optional): Initial comment with file

**Response:**
- `file_id`: ID of the uploaded file
- `permalink`: Permanent link to the file
- `url_private`: Private download URL

## Jira tools

### create_issue

Create a new Jira issue.

**Parameters:**
- `project` (required): Project key or ID
- `issue_type` (required): Type of issue (Bug, Task, Story, etc.)
- `summary` (required): Issue summary/title
- `description` (optional): Issue description
- `priority` (optional): Issue priority
- `assignee` (optional): Assignee email or username
- `labels` (optional): Array of labels
- `components` (optional): Array of components
- `custom_fields` (optional): Custom field values

**Response:**
- `issue_id`: ID of the created issue
- `issue_key`: Key of the created issue (e.g., PROJ-123)
- `issue_url`: URL to view the issue

**Example:**
```javascript
await agentConnect.tools.execute({
  connected_account_id: 'jira_account',
  tool: 'create_issue',
  parameters: {
    project: 'PROJ',
    issue_type: 'Bug',
    summary: 'Login page crashes on mobile Safari',
    description: 'When users try to log in using mobile Safari, the page crashes with a JavaScript error.',
    priority: 'High',
    assignee: 'developer@example.com',
    labels: ['mobile', 'safari', 'login'],
    components: ['Frontend'],
    custom_fields: {
      'environment': 'Production',
      'affects_version': '2.1.0'
    }
  }
});
```

### fetch_issues

Retrieve Jira issues using JQL (Jira Query Language).

**Parameters:**
- `jql` (required): JQL query string
- `max_results` (optional): Maximum number of issues to return
- `start_at` (optional): Starting index for pagination
- `fields` (optional): Fields to include in response
- `expand` (optional): Additional data to expand

**Response:**
- `issues`: Array of issue objects
- `total`: Total number of matching issues
- `start_at`: Starting index of current page

### update_issue

Update an existing Jira issue.

**Parameters:**
- `issue_key` (required): Issue key (e.g., PROJ-123)
- `summary` (optional): New issue summary
- `description` (optional): New issue description
- `priority` (optional): New priority
- `assignee` (optional): New assignee
- `labels` (optional): New labels (replaces existing)
- `custom_fields` (optional): Custom field updates

**Response:**
- `issue_key`: Updated issue key
- `updated_at`: Timestamp when issue was updated

### transition_issue

Change the status of a Jira issue.

**Parameters:**
- `issue_key` (required): Issue key (e.g., PROJ-123)
- `transition` (required): Target status or transition name
- `comment` (optional): Comment to add with transition
- `fields` (optional): Fields to update during transition

**Response:**
- `issue_key`: Transitioned issue key
- `new_status`: New status of the issue
- `transitioned_at`: Timestamp when transition occurred

## Tool execution patterns

### Synchronous execution

Most tools execute synchronously:

```javascript
const result = await agentConnect.tools.execute({
  connected_account_id: 'account_123',
  tool: 'send_email',
  parameters: { /* ... */ }
});
// Result is available immediately
```

### Batch execution

Execute multiple tools in parallel:

```javascript
const results = await agentConnect.tools.executeBatch([
  {
    connected_account_id: 'gmail_account',
    tool: 'send_email',
    parameters: { /* ... */ }
  },
  {
    connected_account_id: 'slack_account',
    tool: 'send_message',
    parameters: { /* ... */ }
  }
]);
```

### Conditional execution

Execute tools based on conditions:

```javascript
const result = await agentConnect.tools.execute({
  connected_account_id: 'account_123',
  tool: 'send_email',
  parameters: { /* ... */ },
  conditions: {
    if_account_active: true,
    if_permissions_granted: ['gmail.send']
  }
});
```

## Common error codes

All tools may return these standard error codes:

- `INVALID_PARAMETERS`: Invalid or missing parameters
- `ACCOUNT_NOT_AUTHENTICATED`: Connected account not authenticated
- `INSUFFICIENT_PERMISSIONS`: Missing required permissions
- `RATE_LIMIT_EXCEEDED`: API rate limit exceeded
- `PROVIDER_ERROR`: Error from the third-party provider
- `NETWORK_ERROR`: Network connectivity issues
- `TIMEOUT`: Request timeout
- `UNKNOWN_ERROR`: Unexpected error occurred

## Provider-specific errors

### Gmail errors

- `INVALID_QUERY`: Invalid Gmail search query
- `MESSAGE_NOT_FOUND`: Requested message not found
- `QUOTA_EXCEEDED`: Gmail quota exceeded
- `ATTACHMENT_TOO_LARGE`: Attachment exceeds size limit

### Slack errors

- `CHANNEL_NOT_FOUND`: Channel does not exist
- `USER_NOT_IN_CHANNEL`: User not member of channel
- `MESSAGE_TOO_LONG`: Message exceeds length limit
- `INVALID_BLOCKS`: Invalid Block Kit blocks

### Jira errors

- `PROJECT_NOT_FOUND`: Project does not exist
- `INVALID_JQL`: Invalid JQL query syntax
- `ISSUE_NOT_FOUND`: Issue does not exist
- `WORKFLOW_VIOLATION`: Invalid status transition

## Tool versioning

Tools are versioned to ensure backward compatibility:

```javascript
// Use specific version
await agentConnect.tools.execute({
  connected_account_id: 'account_123',
  tool: 'send_email',
  version: '1.2.0',
  parameters: { /* ... */ }
});

// Use latest version (default)
await agentConnect.tools.execute({
  connected_account_id: 'account_123',
  tool: 'send_email',
  parameters: { /* ... */ }
});
```

## Getting tool information

Retrieve detailed information about available tools:

```javascript
// List all tools
const allTools = await agentConnect.tools.list();

// List tools for specific provider
const gmailTools = await agentConnect.tools.list({ provider: 'gmail' });

// Get tool definition
const toolDef = await agentConnect.tools.get('send_email');
console.log(toolDef.input_schema);
console.log(toolDef.output_schema);
```
