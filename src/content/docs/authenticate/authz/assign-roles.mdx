After registering roles and permissions for your application, Scalekit provides multiple ways to assign roles to users. These roles allow your app to make the access control decisions as scalekit sends them to your app in the access token.

## Auto assign roles as users join organizations

By default, the organization creator automatically receives the `admin` role, while users who join later receive the `member` role. You can customize these defaults to match your application's security requirements. For instance, in a CRM system, you may want to set the default role for new members to a read-only role like `viewer` to prevent accidental data modifications.

1. Go to **Dashboard** > **Roles & Permissions** > **Roles** tab
2. Select the roles available and choose defaults for organization creator and member

![](@/assets/docs/assign-roles/full-page-highlighth-defaults.png)

This automatically assigns these roles to every users who joins any organization in your Scalekit environment.

## Assign roles using the SDK

For applications that need to manage roles programmatically, use the Scalekit SDK to assign, update, or remove user roles. This is useful for building custom admin interfaces or automated workflows.

<Aside type="caution">
Always verify that the current user has authorization to manage roles before performing role assignments. Implement proper access controls to prevent privilege escalation attacks.
</Aside>

```python title="Assign role to user" wrap collapse={1-12}
import os

        return result

        print(f"Failed to assign role: {error}")
        raise
```

```javascript title="Remove role from user" wrap
// Use case: Remove admin access when user leaves leadership position
async function removeRoleFromUser(userId, organizationId, rolesToRemove) {
  try {
    // Get current membership to see existing roles
    const currentMembership = await scalekit.membership.getMembership({
      user_id: userId,
      organization_id: organizationId

    const updatedRoles = currentMembership.roles.filter(

      roles: updatedRoles
```

```go title="Remove role from user" wrap
// Use case: Clean up roles when user changes departments
func removeRoleFromUser(ctx context.Context, userId, organizationId string, rolesToRemove []string) error {
    // Get current membership to see existing roles
    currentMembership, err := sc.Membership().GetMembership(ctx, &scalekit.GetMembershipRequest{
        UserId:         userId,
        OrganizationId: organizationId,
    })

    if err != nil {
        log.Printf("Failed to get current membership: %v", err)
        return err
    }

    // Filter out the roles to remove
    var updatedRoles []string
    for _, role := range currentMembership.Roles {
        shouldKeep := true
        for _, removeRole := range rolesToRemove {
            if role == removeRole {
                shouldKeep = false
                break
            }
        }
        if shouldKeep {
            updatedRoles = append(updatedRoles, role)
        }
    }

    // Update membership with remaining roles
    _, err = sc.Membership().UpdateMembership(ctx, &scalekit.UpdateMembershipRequest{
        UserId:         userId,
        OrganizationId: organizationId,
        Roles:          updatedRoles,
    })

    if err != nil {
        log.Printf("Failed to remove role: %v", err)
        return err
    }

    log.Printf("Role removed successfully from user: %s", userId)
    return nil
}
```

```java title="Remove role from user" wrap
// Initialize additional imports
import java.util.stream.Collectors;

// Use case: Revoke access when employee terminates employment
public void removeRoleFromUser(String userId, String organizationId, List<String> rolesToRemove) {
    try {
        // Get current membership to see existing roles
        GetMembershipRequest getRequest = GetMembershipRequest.newBuilder()
            .setUserId(userId)
            .setOrganizationId(organizationId)
            .build();

        Membership currentMembership = scalekitClient.memberships().getMembership(getRequest);

        // Filter out the roles to remove
        List<String> updatedRoles = currentMembership.getRolesList().stream()
            .filter(role -> !rolesToRemove.contains(role))
            .collect(Collectors.toList());

        // Update membership with remaining roles
        UpdateMembershipRequest updateRequest = UpdateMembershipRequest.newBuilder()
            .setUserId(userId)
            .setOrganizationId(organizationId)
            .addAllRoles(updatedRoles)
            .build();

        UpdateMembershipResponse response = scalekitClient.memberships().updateMembership(updateRequest);
        System.out.println("Role removed successfully from user: " + userId);

    } catch (Exception e) {
        System.err.println("Failed to remove role: " + e.getMessage());
        throw new RuntimeException("Role removal failed", e);
    }
}
```

## Role assignment methods

Scalekit supports several approaches to role assignment:

- **Dashboard management**: Update user roles directly in the Scalekit dashboard for quick changes and initial setup
- **Default role configuration**: Set automatic role assignments for new organization members
- **Directory group-based assignment (SCIM)**: Map directory groups to application roles for automated provisioning
- **SDK-powered role changes**: Build in-app role management for authorized administrators

## Assign a role in the Scalekit dashboard

![Change a user's role in the Scalekit dashboard](@/assets/docs/assign-roles/change-role.png)


Use these steps to change a user's role from the dashboard.

<Steps>
1. Navigate to **Dashboard** > **Organizations** > **Select organization**
2. In the **Users** tab, you will see list of users.
3. Click "..." and select "Change role"
4. This should ideally open a modal to select the role you want to assign to the user.
</Steps>


To change the default role assignments for new members who join the organization:

![Configure default roles for creators and members](@/assets/docs/assign-roles/change-default-roles.png)

<Steps>
1. Go to **Dashboard** > **Roles & Permissions** > **Roles** tab
2. Click on the role
3. Select the roles available and choose defaults for organization creator and member.
</Steps>

<Aside type="tip">
Default roles help you set safe, least-privilege access for new members while keeping creators fully empowered to configure the workspace.
</Aside>

## Automate role assignment with directory groups (SCIM)

Use SCIM provisioning to map directory provider groups to your app roles. When administrators add users to groups in their directory, Scalekit sends events so your app can update roles automatically. This reduces manual changes and keeps access aligned with enterprise policies.

See the full guide at [Group-based role assignment](/directory/guides/group-based-role-assignment/).

## Best practices for role management

- **Principle of least privilege**: Assign users only the minimum roles they need to perform their job functions
- **Regular audits**: Periodically review user roles and remove unnecessary access
- **Document role purposes**: Clearly define what each role allows users to do
- **Use default roles**: Configure appropriate default roles for new members to maintain security hygiene
- **Monitor role changes**: Log and review role assignment changes for security compliance

<Aside type="tip">
Consider implementing role assignment workflows that require approval for sensitive roles like `admin` or `owner` to maintain proper access controls.
</Aside>
