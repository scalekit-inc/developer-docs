Use this guide when you validate Scalekit **ID tokens** in a language or framework without an official Scalekit SDK (for example, Ruby on Rails, PHP, or Elixir). You fetch OpenID metadata, load public signing keys, and verify the JWT signature and required claims.

If you use a Scalekit SDK, token validation is handled automatically after you exchange the authorization code.

## Key endpoints and parameters

| Parameter | Value |
|-----------|-------|
| OpenID configuration | `https://<YOUR_ENVIRONMENT_URL>/.well-known/openid-configuration` |
| Issuer (`iss`) | Your Scalekit environment URL (same as `issuer` in the OpenID document) |
| JWKS URI (`jwks_uri`) | `https://<YOUR_ENVIRONMENT_URL>/keys` |
| Signing algorithm | `RS256` |

Replace `<YOUR_ENVIRONMENT_URL>` with the URL from **Dashboard > Settings > Environment URL**.

## Validation flow

1. Fetch `/.well-known/openid-configuration` and read `issuer` and `jwks_uri`.
2. Fetch the JWKS document from `jwks_uri`.
3. Decode the JWT header and select the key whose `kid` matches.
4. Verify the signature with **RS256**.
5. Validate claims: `iss`, `aud`, and `exp` at minimum.

<Aside type="note" title="Audience must match your client ID">
  The `aud` claim must equal your application's client ID (`skc_...`). A mismatch means the token was issued for a different application and must be rejected.
</Aside>

## Example — Ruby on Rails

This example uses the `jwt` gem. Store your environment URL and client ID in environment variables — never hard-code secrets in source control.

```ruby
require "jwt"
require "net/http"
require "json"

ENV_URL = ENV.fetch("SCALEKIT_ENV_URL") # e.g. https://yourapp.scalekit.com
CLIENT_ID = ENV.fetch("SCALEKIT_CLIENT_ID")

def fetch_openid_config
  uri = URI("#{ENV_URL}/.well-known/openid-configuration")
  JSON.parse(Net::HTTP.get(uri))
end

def fetch_jwks(jwks_uri)
  JSON.parse(Net::HTTP.get(URI(jwks_uri)))
end

def verify_id_token(id_token)
  config = fetch_openid_config
  jwks = fetch_jwks(config["jwks_uri"])

  # jwt gem resolves the correct key from JWKS using the token header's kid
  decoded, = JWT.decode(
    id_token,
    nil,
    true,
    {
      algorithms: ["RS256"],
      jwks: jwks,
      iss: config["issuer"],
      verify_iss: true,
      aud: CLIENT_ID,
      verify_aud: true,
    }
  )

  decoded
rescue JWT::DecodeError => e
  # Security: Reject the request if validation fails; do not create a session.
  raise "Invalid ID token: #{e.message}"
end
```

## Example — manual steps with cURL

Use these commands to confirm issuer and JWKS before you wire validation into your app:

```bash
# Discover issuer and jwks_uri
curl -s "https://<YOUR_ENVIRONMENT_URL>/.well-known/openid-configuration" | jq '{issuer, jwks_uri}'

# Fetch public signing keys
curl -s "https://<YOUR_ENVIRONMENT_URL>/keys" | jq '.keys[] | {kid, alg, use}'
```

## Claims to validate

| Claim | Rule |
|-------|------|
| `iss` | Must equal your Scalekit environment URL |
| `aud` | Must include your client ID |
| `exp` | Must be in the future at validation time |
| `sub` | Stable user identifier for your session store |

For the full claim reference, see [ID token claims](/guides/idtoken-claims/).
