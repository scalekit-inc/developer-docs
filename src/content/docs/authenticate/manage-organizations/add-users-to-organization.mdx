You can add users to organizations through:

- **Dashboard management** - Manual user addition through the Scalekit admin interface
- **Admin invitations** - Programmatic user invitations through your application using SDK/API
- **Domain-based joining** - Automated organization discovery for users with trusted email domains
- **Just-in-time provisioning** - Enterprise automation through SSO and SCIM integration

## Prerequisites

Before adding users to organizations, ensure you have:

1. **Scalekit SDK configured** - Follow the [installation guide](/authenticate/installation/) to set up your development environment
2. **Organizations created** - Review the [create organizations guide](/fsa/guides/create-organization/) to establish your organizational structure
3. **User management basics** - Understand the [manage users guide](/fsa/guides/manage-users/) for user profile fundamentals

<Steps>

1. ## Add users through the Scalekit dashboard

   Use the dashboard for manual user addition when support teams help customers onboard or when you need to seed initial organization members.

   Navigate to **Dashboard** > **Organizations**, select your target organization, and click the **Users** tab.

   **Invite new users:**

   1. Click **Invite User** to open the invitation form
   2. Enter the user's **email address** (must match organization's allowed domains)
   3. Select their **role** (Admin, Member, or custom organizational roles)
   4. Add optional **personal information** (first name, last name, display name)
   5. Click **Send Invitation**

   The user receives an email invitation with a secure link. When they accept:
   - Scalekit creates their user account (if they're new to your system)
   - Adds them to the organization with the specified role
   - Redirects them to your application's callback URL for session creation

   <Aside type="tip">
   Dashboard invitations automatically respect your organization's allowed email domains and signup restrictions configured in [organization settings](/fsa/guides/organization-settings/).
   </Aside>

2. ## Enable admin invitations through your application

   Build invitation capabilities into your application so organization administrators can invite team members without accessing the Scalekit dashboard. This approach gives you full control over the user experience while leveraging Scalekit's invitation infrastructure.

   These examples use the Scalekit client you configured in the [installation guide](/authenticate/installation/).

3. ## Enable domain-based organization discovery

   Configure organizations to automatically suggest membership to users with trusted email domains. This approach reduces friction for corporate teams while maintaining security through domain verification.

   **Configure allowed domains:**

   Set up trusted email domains in [organization settings](/fsa/guides/organization-settings/) to enable automatic discovery:

   1. Navigate to **Dashboard** > **Organizations** > **Security** > **Domains**
   2. Add corporate domains (e.g., `@company.com`, `@subsidiary.co.uk`)
   3. Enable **automatic organization suggestions** for matching domains
   4. Choose approval requirements (automatic joining vs. admin approval)

   **User experience for domain matching:**

   When users sign up with emails matching allowed domains:

   1. **Domain recognition** - Scalekit detects the corporate email domain during signup
   2. **Organization discovery** - Users see a list of organizations they can join based on their domain
   3. **Self-service joining** - Users select their organization and gain access according to your approval settings
   4. **Admin notifications** - Organization admins receive notifications about new members (configurable)

   **Implement domain-based routing:**

   <Tabs syncKey="tech-stack">
     <TabItem value="node" label="Node.js">

   ```javascript title="Check domain availability" wrap collapse={1-2}
   // Use case: Route users to appropriate authentication method based on email domain
   app.post('/auth/check-domain', async (req, res) => {
     const { email } = req.body
     const domain = email.split('@')[1].toLowerCase()

     try {
       // Check if domain has active SSO connections
       const connections = await scalekit.connections.listConnectionsByDomain({
         domain
       })

       if (connections.length > 0) {
         // Domain has SSO configured - redirect to identity provider
         const authUrl = scalekit.getAuthorizationUrl(redirectUri, {
           domainHint: domain  // Automatically routes to correct IdP
         })
         return res.json({ authUrl, method: 'sso' })
       } else {
         // Check for organizations accepting this domain
         const { organizations } = await scalekit.organization.listOrganization({
           allowedDomain: domain
         })

         if (organizations.length > 0) {
           return res.json({ 
             organizations, 
             method: 'domain-join',
             message: 'Organizations available for your domain'
           })
         } else {
           // No organization for this domain - show standard signup
           return res.json({ method: 'standard-signup' })
         }
       }
     } catch (error) {
       res.status(400).json({ error: error.message })
     }
   })
   ```

   // Use case: Route users to appropriate authentication method based on email domain
   func checkDomainAvailability(c *gin.Context) {
       var req struct {
           Email string `json:"email"`
       }
       
       if err := c.ShouldBindJSON(&req); err != nil {
           c.JSON(400, gin.H{"error": err.Error()})
           return
       }
       
       parts := strings.Split(req.Email, "@")
       if len(parts) != 2 {
           c.JSON(400, gin.H{"error": "Invalid email format"})
           return
       }
       domain := strings.ToLower(parts[1])
       
       // Check if domain has active SSO connections
       connections, err := scalekitClient.Connections.ListByDomain(ctx, scalekit.ListConnectionsByDomainRequest{
           Domain: domain,
       })
       
       if err == nil && len(connections) > 0 {
           // Domain has SSO configured - redirect to identity provider
           authUrl, _ := scalekitClient.GetAuthorizationUrl(redirectUri, scalekit.AuthorizationUrlOptions{
               DomainHint: domain, // Automatically routes to correct IdP
           })
           c.JSON(200, gin.H{"auth_url": authUrl.String(), "method": "sso"})
           return
       }
       
       // Check for organizations accepting this domain
       organizations, err := scalekitClient.Organization.List(ctx, scalekit.ListOrganizationsRequest{
           AllowedDomain: domain,
       })
       
       if err == nil && len(organizations.Organizations) > 0 {
           c.JSON(200, gin.H{
               "organizations": organizations.Organizations,
               "method":        "domain-join",
               "message":       "Organizations available for your domain",
           })
       } else {
           // No organization for this domain - show standard signup
           c.JSON(200, gin.H{"method": "standard-signup"})
       }
   }

   // Use case: Route users to appropriate authentication method based on email domain
   @PostMapping("/auth/check-domain")
   public ResponseEntity<Map<String, Object>> checkDomainAvailability(
       @RequestBody Map<String, String> request
   ) {
       String email = request.get("email");
       String domain = email.split("@")[1].toLowerCase();
       
       try {
           // Check if domain has active SSO connections
           List<Connection> connections = scalekitClient.connections()
               .listConnectionsByDomain(domain);
           
           if (!connections.isEmpty()) {
               // Domain has SSO configured - redirect to identity provider
               String authUrl = scalekitClient.authentication()
                   .getAuthorizationUrl(redirectUri, 
                       AuthorizationUrlOptions.builder()
                           .domainHint(domain) // Automatically routes to correct IdP
                           .build())
                   .toString();
               return ResponseEntity.ok(Map.of(
                   "auth_url", authUrl, 
                   "method", "sso"
               ));
           } else {
               // Check for organizations accepting this domain
               ListOrganizationsResponse orgResponse = scalekitClient.organizations()
                   .listOrganizations(ListOrganizationsRequest.builder()
                       .allowedDomain(domain)
                       .build());
               
               if (!orgResponse.getOrganizationsList().isEmpty()) {
                   return ResponseEntity.ok(Map.of(
                       "organizations", orgResponse.getOrganizationsList(),
                       "method", "domain-join",
                       "message", "Organizations available for your domain"
                   ));
               } else {
                   // No organization for this domain - show standard signup
                   return ResponseEntity.ok(Map.of("method", "standard-signup"));
               }
           }
       } catch (Exception error) {
           return ResponseEntity.badRequest().body(Map.of("error", error.getMessage()));
       }
   }

4. ## Set up automated provisioning

   Configure just-in-time (JIT) user provisioning through enterprise identity providers to eliminate manual user management for large organizations.

   **Just-in-time provisioning via SSO** <Badge text="Coming soon" variant="note" />

   When organizations connect their SSO provider (Okta, Azure AD, Google Workspace), enable automatic user creation:

   - **First-time authentication** - New users authenticate through their corporate identity provider
   - **Automatic account creation** - Scalekit creates user accounts with SSO profile data  
   - **Organization assignment** - Users join organizations based on identity provider claims
   - **Role mapping** - SSO attributes determine initial roles and permissions

   **Just-in-time provisioning via SCIM** <Badge text="Coming soon" variant="note" />

   SCIM (System for Cross-domain Identity Management) provides real-time user synchronization:

   - **Automated synchronization** - Identity providers push user changes to Scalekit automatically
   - **Lifecycle management** - Handle user creation, updates, deactivation, and deletion
   - **Group mapping** - Sync organizational units and teams from corporate directories
   - **Bulk operations** - Efficiently manage large user populations without manual intervention

   Both SSO and SCIM provisioning configurations are managed through [organization settings](/fsa/guides/organization-settings/) and require enterprise identity provider setup by your customers.

## Security and access controls

Implement security measures across all user addition methods to prevent unauthorized access and maintain data integrity.

**Recommended security practices:**

- **Email verification** - Always verify email addresses before granting organization access
- **Domain verification** - Confirm organizational ownership of email domains through DNS records
- **Role-based permissions** - Validate that requesting users have admin rights before processing invitations
- **Rate limiting** - Prevent abuse of invitation and signup endpoints
- **Audit logging** - Track all user addition activities for compliance and security monitoring

**Access control patterns:**

```javascript title="Example: Role-based invitation validation" showLineNumbers=false
// Verify admin permissions before allowing invitations

  req.session.userId, 
  orgId,

)

  return res.status(403).json({ error: 'Insufficient permissions' })
}
```

Review the [signup restrictions guide](/fsa/guides/signup-restrictions/) for additional security controls and the [organization settings guide](/fsa/guides/organization-settings/) for enterprise configuration options.
