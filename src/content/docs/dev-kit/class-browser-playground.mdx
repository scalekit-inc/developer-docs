## How to use this page

1. **Left sidebar** — manual Starlight config (`sidebar.config.ts`). Only what we list appears.
2. **Right TOC** — only **markdown headings** (`##` / `###` / `####`) on this page. ClassBrowser
   method names do **not** auto-appear in the TOC.
3. **In-component anchors** — each `ClassMethod` gets `id={methodName}`. Hash links like
   `#greet` open/scroll that method. Those anchors are **invisible to the right TOC** unless you
   also add a matching markdown heading.
4. **Header tools** on each browser — Copy as JSON, expand/collapse all methods.

## Navigation matrix (what works / does not)

| Surface | What drives it | Appears for ClassBrowser methods? | Notes |
| --- | --- | --- | --- |
| **Left sidebar** | `sidebar.config.ts` (or autogenerate dirs) | Only if you add explicit links / pages | Best for language → client → page splits |
| **Right TOC** | Markdown `##` / `###` / `####` | **No** (methods are `<details>`, not headings) | Headings around browsers *do* show |
| **In-page hash** | `ClassMethod` `id={name}` | Yes, if you link `#methodName` | Case-sensitive; dots in names become awkward IDs |
| **Browser chrome** | Expand all / Copy JSON | N/A | Local to each `ClassBrowser` card |
| **`source` text** | `source` prop | Display only (not always a link) | Prefer full GitHub URL for humans |
| **`target` link** | `target` prop / TargetLink | Small chevron link | Good for “see also” |

### Patterns to try for SDK reference

| Pattern | Left sidebar | Right TOC | Verdict to evaluate |
| --- | --- | --- | --- |
| **A. One page / language** (current) | Language only | Only section headings you write | TOC thin; long scroll |
| **B. One page / client** | Language → Actions, Tools, Orgs… | Client page short; method TOC still missing | Sidebar does the work |
| **C. Headings per method** | Language or client | You duplicate `### methodName` above each method | TOC dense; maintenance heavy |
| **D. Hybrid** | Language → client pages | `##` groups (Auth, Accounts…) + ClassBrowser under each | Likely best balance |

### 6. What ClassBrowser does **not** do

- No built-in multi-language tabs (use Starlight `Tabs` outside the browser).
- No automatic sidebar generation from methods.
- No automatic right-TOC entries for methods (unless you add markdown headings).
- No OpenAPI import — hand-authored (or your own generator) only.
- `source` is display text in the header, not always a clickable link (use `target` for links).
- Method `name` becomes the HTML `id` — avoid spaces; be careful with `.` in names for hash links.
