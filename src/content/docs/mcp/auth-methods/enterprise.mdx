<Subtitle>Authenticate your MCP server with Enterprise SSO in minutes.</Subtitle>

Scalekit supports a variety of Enterprise SSO options, such as Okta, Microsoft Entra ID, Google Workspace, JumpCloud, Microsoft ADFS, OneLogin, Ping Identity, Shibboleth, and any SAML or OIDC provider. This section focuses on you can configure **Okta OIDC** for your MCP server. The same steps would apply for any other Enterprise SSO.

## Before you start

- A Scalekit workspace (https://app.scalekit.com).
- Basic understanding of MCP Auth with examples [link](/mcp/integrations/fastmcp)

## Configure OKTA for authentication

1. In the [Scalekit dashboard](https://app.scalekit.com) -> Organizations (left nav) -> Create Organization 

![Create Organization](@/assets/docs/guides/mcp/create-org.png)

2. Fill in the **Single Sign-on Details -> Test Connection -> Enable Connection**

![Setup Organization SSO](@/assets/docs/guides/mcp/setup-org-sso.png)

3. Add the domain for which you want authentication to be done using Okta (eg. acmecorp.com).

![Organization Domain Setup](@/assets/docs/guides/mcp/org-domain.png)

## How to make your MCP server use this connection
Scalekit automatically handles different connections configured by you. You dont need to do any code change. Run any of the [example apps](/mcp/integrations/fastmcp) and when prompted to login, login using any email address with the configured domain.
