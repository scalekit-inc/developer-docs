## Overview

Passwordless authentication eliminates the need for traditional passwords by using alternative verification methods. This improves security and user experience.

## Available Methods

### Magic Links

Send a unique link to the user's email that automatically logs them in when clicked.

**Pros:**
- Simple user experience
- No password to remember
- Works across devices

**Cons:**
- Requires email access
- Can be slower than other methods

[Learn more about magic links →](/passwordless/quickstart)

### One-Time Passwords (OTP)

Send a temporary code to the user's email or phone that they enter to authenticate.

**Pros:**
- Familiar to users
- Works offline after code is received
- Can use SMS or email

**Cons:**
- Requires manual code entry
- SMS can have delivery issues

[Learn more about OTP →](/passwordless/oidc)

### Biometric Authentication

Use fingerprint, face recognition, or other biometric data for authentication.

**Pros:**
- Very fast authentication
- Excellent security
- Great user experience

**Cons:**
- Requires compatible hardware
- Not available on all devices

[Learn about Passkeys (coming soon) →](/authenticate/passkeys)

## Implementation

<Tabs>
  <TabItem label="Magic Links">
    ```javascript
    // Initialize magic link authentication
    const scalekit = new Scalekit({
      clientId: process.env.SCALEKIT_CLIENT_ID,
      clientSecret: process.env.SCALEKIT_CLIENT_SECRET,
    })

    // Send magic link
    await scalekit.passwordless.sendMagicLink({
      email: 'user@example.com',
      redirectUrl: 'https://yourapp.com/auth/callback'
    })
    ```
  </TabItem>

  <TabItem label="OTP">
    ```javascript
    // Send OTP
    await scalekit.passwordless.sendOTP({
      email: 'user@example.com',
      method: 'email' // or 'sms'
    })

    // Verify OTP
    const result = await scalekit.passwordless.verifyOTP({
      email: 'user@example.com',
      code: '123456'
    })
    ```
  </TabItem>
</Tabs>

## Best Practices

1. **Link Expiration**: Set appropriate expiration times for magic links (typically 15-30 minutes)
2. **Rate Limiting**: Prevent abuse by limiting requests per user
3. **Fallback Options**: Provide alternative login methods
4. **Clear Communication**: Inform users where to find their authentication code/link
5. **Email Deliverability**: Use a reliable email provider to ensure delivery
