## Overview

OAuth-driven SSO allows you to implement Single Sign-On using OAuth 2.0 flows. This approach is commonly used for social logins and modern identity providers.

## OAuth Flow for SSO

<Steps>

1. **User initiates login**

   The user clicks on a social login button or enterprise SSO option.

2. **Authorization request**

   Your application redirects the user to the OAuth authorization endpoint.

3. **User authentication**

   The identity provider authenticates the user and requests consent.

4. **Authorization code**

   The IdP redirects back to your application with an authorization code.

5. **Token exchange**

   Your application exchanges the authorization code for access and ID tokens.

6. **User session**

   Create a session for the authenticated user in your application.

</Steps>

## Best Practices

<Card title="Security Considerations" icon="warning">
  - Always validate the `state` parameter to prevent CSRF attacks
  - Use PKCE (Proof Key for Code Exchange) for mobile and SPA applications
  - Validate ID tokens before creating user sessions
  - Implement proper redirect URI validation
</Card>
