Adding [Agent Auth](/agentkit/overview/) to an application by hand means writing OAuth (Open Authorization) callback handlers, a token vault, a refresh scheduler, and connector-specific API clients — before you write a single line of agent logic. Scalekit eliminates that overhead: it manages the full OAuth lifecycle and exposes a single API surface regardless of which connector your agent calls.

This recipe uses a Scalekit Auth Stack plugin to let your coding agent analyze your codebase and generate production-ready Agent Auth code in under two minutes.

## What this recipe covers

- Installing the Scalekit Auth Stack into Claude Code, Codex, GitHub Copilot CLI, Cursor, or any of 40+ agents via the Vercel Skills CLI
- Using a structured prompt so the agent generates client initialization, connected account management, OAuth authorization links, token fetching, and refresh logic
- Verifying that the generated code handles the OAuth flow end-to-end before you ship

What the recipe does **not** cover: configuring connectors in the Scalekit Dashboard (see "Before you start") or customizing the generated code for your specific framework.

## Before you start

You need three things in place before running the plugin install:

- **A Scalekit account** — sign up at [app.scalekit.com](https://app.scalekit.com) if you haven't already.
- **A connector configured** — in the dashboard, open **Agent Auth → Connections** and create a connection for the service you want to call (for example, `gmail` or `googlecalendar`). The connection name is what your generated code references.
- **Your coding agent installed** — pick one tab below and ensure the agent is running before proceeding.

## Verify the setup

Once the coding agent finishes generating code, confirm the full OAuth flow works end-to-end:

1. **Check generated files** — your project should now include a Scalekit client initialization file, a connected account helper, and an authorization URL generator. Each agent's verify step above shows exactly what to look for.
2. **Set environment variables** — copy the `.env.example` the agent generated (if present) and fill in your `SCALEKIT_CLIENT_ID`, `SCALEKIT_CLIENT_SECRET`, and `SCALEKIT_ENVIRONMENT_URL` from the [Scalekit Dashboard](https://app.scalekit.com) → **API Credentials**.
3. **Trigger the authorization flow** — run the generated example (or follow the agent's instructions) and confirm the browser redirects to the connector's consent page.
4. **Fetch a token** — after consent, call the token-fetch function and confirm you receive a valid access token. The Scalekit client handles refresh automatically from this point on.

<Aside type="caution" title="Review generated code before deploying">
  AI-generated authentication code requires a security review. Verify that token validation logic, error handling for expired tokens, and environment variable references all match your application's requirements. The generated code is a foundation, not a finished implementation.
</Aside>

## Common scenarios

<details>
<summary>The agent generated code for a connector I haven't configured yet</summary>

The plugin uses the connector name you provide in the prompt. If that connector isn't configured in your Scalekit Dashboard, the OAuth flow will fail at runtime with a "connector not found" error.

Fix: go to **Agent Auth → Connections** in the [Scalekit Dashboard](https://app.scalekit.com), create the connection, then re-run the agent prompt with the exact connection name from the dashboard.

</details>

<details>
<summary>I want to swap connectors after the initial generation</summary>

Re-run the implementation prompt with the new connector name. The agent updates the connector reference in the client initialization and regenerates the token-fetch call. Existing connected accounts for the old connector are not affected.

</details>

<details>
<summary>The scaffolded code references an SDK version that doesn't match my lockfile</summary>

The plugin targets the latest stable Scalekit SDK. If your lockfile pins an older version, either upgrade the SDK (`npm install @scalekit-sdk/node@latest` or equivalent) or ask the agent to regenerate using your pinned version by adding "use SDK version X.Y.Z" to the prompt.

</details>

<Aside type="tip" title="Explore available connectors">
  Browse [Scalekit's connector library](https://app.scalekit.com) → **Agent Auth** to see all available services you can integrate with your AI agents. New connectors are added regularly.
</Aside>
