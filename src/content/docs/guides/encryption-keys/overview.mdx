Scalekit protects your sensitive data using a Scalekit managed Data Encryption Key (DEK). Every environment gets its own key, so data is always isolated across environments. If your compliance or security policy requires it, you can bring your own key from a KMS system instead.

## Key types

| Type | Description |
|------|-------------|
| **Scalekit managed DEK** | Scalekit generates and manages the key. No setup required. |
| **Bring your own key (BYOK)** | You provide a key from Google Cloud KMS. You own the key lifecycle, including rotation and revocation. |

## Key states

Every encryption key moves through these states:

| State | Meaning |
|-------|---------|
| **Staged** <Badge text="Staged" variant="caution" /> | Created, not yet in use. No data is encrypted with it. |
| **Primary** <Badge text="Primary" variant="success" /> | The active key. All new encryption operations use it. |
| **Deprecated** <Badge text="Deprecated" variant="default" /> | Replaced by a newer key. Older records may still reference it until re-encryption completes. |

## Manage encryption keys

Go to **Settings** and open the **Encryption keys** tab.

### Create a key

Click **Create New Key** and select a provider:

- **Scalekit Managed DEK**: Scalekit generates and manages the key. No further input required.
- **BYOK - GCP Cloud KMS**: The dashboard shows you the Scalekit service account and the IAM roles to grant. Enter the GCP key resource name and verify connectivity inline. See [BYOK: GCP KMS setup](/guides/encryption-keys/byok-gcp/) for the full walkthrough.

The key is created in **Staged** state.

<Aside type="note" title="One primary key at a time">
  Creating a new key does not replace the current Primary key. The new key stays Staged until you explicitly activate it.
</Aside>

### Activate a key

Click **Activate** on a staged key. The key becomes Primary and Scalekit uses it to encrypt all data from that point forward.

### Re-encrypt data

Activating a new key protects new writes. Existing records remain encrypted with the previous key until you re-encrypt them.

Click **Re-encrypt Data**. Scalekit decrypts each existing record using the previous key and re-encrypts it with the new key. The dashboard shows progress and reports any records that could not be recovered.
