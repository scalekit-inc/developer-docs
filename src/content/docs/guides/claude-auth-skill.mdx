The Claude Auth Skill accelerates Scalekit authentication implementation by generating complete, production-ready code for your application. This guide shows you how to install and use the skill to implement authentication in minutes instead of hours.

This skill provides three implementation paths: Full-Stack Authentication for complete auth systems, Modular SSO for adding enterprise authentication to existing setups, and MCP Server Authentication for securing Model Context Protocol servers. After completing this guide, you'll have working authentication code tailored to your framework with security defaults, error handling, and production checklists included.

## Overview

Claude Auth Skill is an AI-powered code generation tool that creates complete Scalekit authentication implementations. The skill automatically detects your technology stack and generates framework-specific code with all necessary imports, environment configuration, and security best practices.

### Key capabilities

- **Framework-aware generation** - Detects your stack and provides complete code for Node.js/Express, Next.js, or Python/FastAPI
- **Security by default** - Implements HttpOnly cookies, CSRF protection, token validation, and short token lifetimes
- **Complete implementations** - Generates working code with imports, dependencies, environment setup, and error handling
- **Three authentication paths** - Supports full-stack auth, modular SSO, and MCP server authentication
- **Production-ready** - Includes validation scripts, production checklists, and deployment guidance

### When to use this skill

Use Claude Auth Skill when you need to:

- Implement authentication quickly without sacrificing security or code quality
- Add Scalekit authentication to a new application
- Integrate enterprise SSO into an existing authentication system
- Secure an MCP server with OAuth 2.1
- Generate framework-specific authentication code

## Prerequisites

Before installing Claude Auth Skill, ensure you have:

- **Python 3.7+** installed on your system
- **Anthropic API key** with access to Claude API or Claude Code
- **Scalekit account** - Sign up at [scalekit.com](https://scalekit.com) to obtain API credentials
- **pip** package manager for Python dependencies

Install required Python packages:

```bash frame="terminal"
pip install anthropic python-dotenv
```

## Use Claude Auth Skill

After installation, activate the skill in your Claude conversation to generate authentication code.

<Steps>

1. ## Start a Claude conversation

   Open Claude API or Claude Code and begin a conversation in your project directory.

   <Aside type="tip" title="Framework detection">
     The skill automatically detects your framework from package.json, requirements.txt, or go.mod files. Ensure these files exist in your project directory for accurate framework detection.
   </Aside>

2. ## Activate the skill

   In your Claude conversation, request authentication implementation using natural language.

   **Example prompts:**

   - "I need to implement full-stack authentication with Scalekit for my Express.js application"
   - "Add enterprise SSO to my existing Next.js app using Scalekit"
   - "Help me secure my MCP server with OAuth 2.1 using Scalekit"

   The skill will activate automatically when you mention authentication implementation.

3. ## Choose your implementation path

   Claude will ask which authentication path you need. Select one of three options:

   <CardGrid>
     <Card title="Full-Stack Authentication" icon="star">
       Complete authentication system for new applications. Includes sign-up, login, session management, social login, and enterprise SSO.

       **Best for:** New applications or complete auth system replacements
     </Card>

     <Card title="Modular SSO" icon="puzzle">
       Adds SAML/OIDC enterprise SSO to existing authentication systems without migration. Keeps your current user database and login logic.

       **Best for:** B2B SaaS adding enterprise features to existing auth
     </Card>

     <Card title="MCP Server Authentication" icon="rocket">
       Implements OAuth 2.1 for securing Model Context Protocol servers with scope-based permissions and token validation.

       **Best for:** MCP servers requiring user authentication
     </Card>
   </CardGrid>

4. ## Review generated code

   Claude will generate complete, framework-specific code including:

   - All necessary imports and dependencies
   - Environment variable configuration
   - Route handlers with security defaults
   - Session management logic
   - Error handling
   - Production deployment checklist

   The generated code includes security features by default:

   - HttpOnly cookies for token storage (prevents XSS attacks)
   - CSRF protection with sameSite settings
   - Token validation on every request
   - Short token lifetimes

   <Aside type="caution">
     Review the generated code carefully before deployment. While the skill generates production-ready code, you should verify it matches your specific security requirements and business logic.
   </Aside>

5. ## Configure environment variables

   Create a `.env` file in your project root with your Scalekit credentials.

   ```bash title=".env" showLineNumbers=false
   SCALEKIT_ENVIRONMENT_URL=https://your-subdomain.scalekit.dev
   SCALEKIT_CLIENT_ID=your_client_id
   SCALEKIT_CLIENT_SECRET=your_client_secret
   ```

   Find these values in **Dashboard > Developers > Settings > API Credentials** in your Scalekit dashboard.

   <Aside type="danger">
     Never commit `.env` files to version control. Add `.env` to your `.gitignore` file to prevent accidental exposure of credentials.
   </Aside>

6. ## Validate your setup

   Use the included validation scripts to verify your configuration before deployment.

   ```bash frame="terminal"
   # Validate environment variables
   python validate_env.py

   # Test connection to Scalekit
   python test_connection.py

   # Test complete authentication flow
   python test_auth_flow.py
   ```

   These scripts verify your environment configuration and confirm successful connection to Scalekit APIs.

</Steps>

## Troubleshooting

### Skill not activating

**Symptom:** Claude doesn't recognize authentication implementation requests

**Resolution:** Ensure the skill is installed correctly by checking the skills directory. Reinstall using the installation script if needed. Verify you're using Claude API or Claude Code (not Claude.ai web interface).

### Framework not detected

**Symptom:** Claude asks which framework you're using instead of auto-detecting

**Resolution:** Ensure your project has framework-specific files:
- Node.js: `package.json` with dependencies
- Python: `requirements.txt` or `pyproject.toml`
- Go: `go.mod` file

### Generated code has errors

**Symptom:** TypeScript compilation errors or runtime errors in generated code

**Resolution:** Verify you have the correct framework version. The skill generates code for current LTS versions. Check that all required dependencies are installed. Review the production checklist included in the generated code.

### Environment validation fails

**Symptom:** `validate_env.py` reports missing or invalid credentials

**Resolution:** Verify your `.env` file contains all required variables: `SCALEKIT_ENVIRONMENT_URL`, `SCALEKIT_CLIENT_ID`, and `SCALEKIT_CLIENT_SECRET`. Confirm these values match your Scalekit dashboard configuration exactly.

## FAQs

**Q: Does the Claude Auth Skill work with Claude.ai web interface?**

A: No. The skill requires Claude API or Claude Code. It cannot be used with the claude.ai web interface.

**Q: Can I customize the generated code?**

A: Yes. The generated code is complete and working but designed to be customized. Modify route paths, add custom validation logic, or integrate with your existing database schema as needed.

**Q: How accurate is the generated code?**

A: Users report 95% code accuracy with no modifications required for basic implementations. You may need to adjust for specific business logic or integrate with existing systems.

**Q: Does the skill support multiple authentication methods?**

A: Yes. The Full-Stack Authentication path supports email/password, social login (Google, GitHub, Microsoft), enterprise SSO (SAML/OIDC), magic links, and passkeys depending on your configuration.

**Q: How long does implementation typically take?**

A: Users report 80% faster implementation compared to manual coding. Typical 4-hour authentication setups reduce to approximately 45 minutes with the skill.

**Q: Is the generated code production-ready?**

A: Yes. The code includes security defaults, error handling, and production checklists. However, you should review and test the code thoroughly before deploying to production. Use the included validation scripts to verify your setup.

**Q: Can I use this skill for multiple projects?**

A: Yes. Workspace/global installation makes the skill available across all Claude conversations. For project-specific needs, use local installation in each project directory.
