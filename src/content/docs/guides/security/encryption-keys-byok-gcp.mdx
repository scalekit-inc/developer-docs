This guide walks you through creating a Google Cloud KMS key and connecting it to your Scalekit environment as a BYOK encryption key. If you are not sure whether BYOK is right for your needs, see [Encryption keys](/guides/security/encryption-keys/) first.

<Aside type="note" title="GCP only (preview)">
  BYOK currently supports Google Cloud KMS. AWS KMS and Azure Key Vault support are planned for a future release. This feature is in preview — contact your Scalekit account team to enable it.
</Aside>

## Prerequisites

- A **Google Cloud project** with the [Cloud KMS API enabled](https://cloud.google.com/kms/docs/create-encryption-keys)
- **IAM permissions** to create key rings, create keys, and set key-level IAM policies (`roles/cloudkms.admin` or equivalent)
- The **Scalekit service account email** — shown in the dashboard when you select BYOK as the key type (see [step 4](#step-4-register-the-byok-key-in-scalekit))
- `gcloud` CLI installed and authenticated (for the CLI approach), or access to the [GCP Console](https://console.cloud.google.com/security/kms)

## Set up BYOK

### Step 1: Create a key ring

A key ring groups related keys in a specific location. Key rings are permanent — they cannot be deleted or renamed.

<Tabs syncKey="gcp-method">
<TabItem label="gcloud CLI">

```bash frame="terminal"
gcloud kms keyrings create "my-scalekit-keyring" \
  --location "global" \
  --project "YOUR-GCP-PROJECT"
```

Replace `YOUR-GCP-PROJECT` with your Google Cloud project ID.

</TabItem>
<TabItem label="GCP Console">

1. Open [**Security > Key Management**](https://console.cloud.google.com/security/kms) in the GCP Console.
2. Click **Create key ring**.
3. Enter a **Key ring name** (for example, `my-scalekit-keyring`).
4. Set **Location type** to **Global** (or choose a region that matches your data-residency requirements).
5. Click **Create**.

</TabItem>
</Tabs>

<Aside type="tip" title="Choose the right location">
  Use `global` if your Scalekit environment has no specific regional data-residency requirement. For region-specific compliance, choose a regional location such as `us-east1` or `europe-west1`. See [Cloud KMS locations](https://cloud.google.com/kms/docs/locations) for the full list.
</Aside>

### Step 2: Create an encryption key

Create a symmetric encryption key inside the key ring.

<Tabs syncKey="gcp-method">
<TabItem label="gcloud CLI">

```bash frame="terminal"
gcloud kms keys create "my-scalekit-key" \
  --location "global" \
  --keyring "my-scalekit-keyring" \
  --purpose "encryption" \
  --project "YOUR-GCP-PROJECT"
```

The `--purpose "encryption"` flag creates an AES-256-GCM symmetric key suitable for envelope encryption.

</TabItem>
<TabItem label="GCP Console">

1. On the **Key Management** page, click the name of the key ring you just created.
2. Click **Create key**.
3. Enter a **Key name** (for example, `my-scalekit-key`).
4. Set **Purpose** to **Symmetric encrypt/decrypt**.
5. Leave **Algorithm** as the default (**Google symmetric key** — AES-256-GCM).
6. Set a **Rotation period** that suits your policy (default is 90 days).
7. Click **Create**.

</TabItem>
</Tabs>

### Step 3: Grant Scalekit access to the key

Grant the Scalekit service account two IAM roles, scoped to the specific key (not the whole project):

| Role | Purpose |
|------|---------|
| `roles/cloudkms.cryptoKeyEncrypterDecrypter` | Encrypt and decrypt the DEK |
| `roles/cloudkms.viewer` | Read key metadata for health checks and listing |

Replace `SCALEKIT-SERVICE-ACCOUNT` with the service account email shown in the Scalekit dashboard.

<Tabs syncKey="gcp-method">
<TabItem label="gcloud CLI">

```bash frame="terminal"
# Grant encrypt/decrypt
gcloud kms keys add-iam-policy-binding "my-scalekit-key" \
  --keyring "my-scalekit-keyring" \
  --location "global" \
  --project "YOUR-GCP-PROJECT" \
  --member "serviceAccount:SCALEKIT-SERVICE-ACCOUNT" \
  --role "roles/cloudkms.cryptoKeyEncrypterDecrypter"

# Grant viewer
gcloud kms keys add-iam-policy-binding "my-scalekit-key" \
  --keyring "my-scalekit-keyring" \
  --location "global" \
  --project "YOUR-GCP-PROJECT" \
  --member "serviceAccount:SCALEKIT-SERVICE-ACCOUNT" \
  --role "roles/cloudkms.viewer"
```

</TabItem>
<TabItem label="GCP Console">

Repeat these steps for each role (`Cloud KMS CryptoKey Encrypter/Decrypter` and `Cloud KMS Viewer`):

1. Go to **Security > Key Management**, click the key ring, then click the key name.
2. Click the **Permissions** tab.
3. Click **Grant access**.
4. In the **New principals** field, enter the Scalekit service account email.
5. In **Select a role**, search for and choose the role:
   - First pass: **Cloud KMS CryptoKey Encrypter/Decrypter**
   - Second pass: **Cloud KMS Viewer**
6. Click **Save**.

<Aside type="note" title="Key-level vs project-level IAM">
  Grant these roles at the **key level** (not at the project level) to limit Scalekit's access to only this specific key.
</Aside>

</TabItem>
</Tabs>

### Step 4: Register the BYOK key in Scalekit

Once IAM is configured, add the key to your Scalekit environment:

1. In the Scalekit dashboard, go to **Configuration > Encryption Keys**.
2. Click **Create key**.
3. Select **BYOK** as the key type. The Scalekit service account email for your environment is displayed — use it in [step 3](#step-3-grant-scalekit-access-to-the-key) above.
4. Select **GCP** as the provider.
5. Enter the fully-qualified GCP key resource name:

   ```
   projects/YOUR-GCP-PROJECT/locations/global/keyRings/my-scalekit-keyring/cryptoKeys/my-scalekit-key
   ```

   To retrieve the exact name from the CLI:

   ```bash frame="terminal"
   gcloud kms keys describe "my-scalekit-key" \
     --keyring "my-scalekit-keyring" \
     --location "global" \
     --project "YOUR-GCP-PROJECT" \
     --format="value(name)"
   ```

6. Click **Create**. The key is created in **Staged** state.

### Step 5: Activate the key

A staged key does not encrypt any data yet. Activate it to make it the active encryption key for your environment:

1. In the **Encryption Keys** table, find the staged key.
2. Open the action menu (⋯) and select **Activate**.

<Aside type="caution" title="Activation is permanent">
  Once activated, you cannot deactivate the key without creating and activating a new key. Confirm your IAM grants are in place before activating.
</Aside>

### Step 6: Re-encrypt existing data

Activating a key protects new writes only. To migrate all existing records to the new key, open the action menu and select **Re-encrypt data**. The dashboard shows the number of records re-encrypted and any rows that could not be recovered.

## Verify the setup

Before activating in production, confirm that Scalekit can perform encrypt/decrypt operations impersonating the Scalekit service account. Run this from Google Cloud Shell:

```bash frame="terminal"
echo -n "ping" | gcloud kms encrypt \
  --key "my-scalekit-key" \
  --keyring "my-scalekit-keyring" \
  --location "global" \
  --project "YOUR-GCP-PROJECT" \
  --plaintext-file - \
  --ciphertext-file - \
  --impersonate-service-account SCALEKIT-SERVICE-ACCOUNT \
  | base64
```

A successful run returns a base64-encoded ciphertext string. A `PERMISSION_DENIED` error means the IAM bindings from step 3 are missing or incorrect.

## Monitor with audit logs

Cloud KMS logs every cryptographic operation to [Cloud Audit Logs](https://cloud.google.com/kms/docs/audit-logging). Use this filter in **Cloud Logging** to monitor Scalekit's key usage:

```
resource.type="cloudkms_cryptokey"
```

This query returns all key management and data access events — encrypt, decrypt, and key update operations — for your KMS keys.

## Key resource name format

The key reference you enter in Scalekit must be a fully-qualified GCP resource name:

```
projects/{PROJECT_ID}/locations/{LOCATION}/keyRings/{KEYRING_NAME}/cryptoKeys/{KEY_NAME}
```

To look up the resource name of your key ring:

```bash frame="terminal"
gcloud kms keyrings describe "my-scalekit-keyring" \
  --location "global" \
  --project "YOUR-GCP-PROJECT" \
  --format="value(name)"
```

Append `/cryptoKeys/my-scalekit-key` to get the full key reference.

## Troubleshooting

<details>
<summary>Permission denied when activating or using the key</summary>

Check that both IAM bindings are applied to the correct key:

```bash frame="terminal"
gcloud kms keys get-iam-policy "my-scalekit-key" \
  --keyring "my-scalekit-keyring" \
  --location "global" \
  --project "YOUR-GCP-PROJECT"
```

The output should list the Scalekit service account with both `roles/cloudkms.cryptoKeyEncrypterDecrypter` and `roles/cloudkms.viewer`. If the bindings are missing, repeat [step 3](#step-3-grant-scalekit-access-to-the-key).

</details>

<details>
<summary>Re-encryption reports unrecoverable rows</summary>

Unrecoverable rows are records that could not be re-encrypted, typically because the previous key version was disabled or destroyed before re-encryption completed. These rows remain encrypted under the prior key version.

Contact [Scalekit support](https://scalekit.com/contact) if you need assistance recovering affected records.

</details>

<details>
<summary>Key resource name is not accepted</summary>

Verify the format is exactly:

```
projects/PROJECT_ID/locations/LOCATION/keyRings/KEYRING_NAME/cryptoKeys/KEY_NAME
```

Run `gcloud kms keys describe` with `--format="value(name)"` to retrieve the exact string — do not construct it manually.

</details>
