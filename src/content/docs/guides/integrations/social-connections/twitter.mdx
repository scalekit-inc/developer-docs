Scalekit enables apps to easily let users sign in using Twitter (X) as their social connector. This guide walks you through the process of setting up the connection between Scalekit and Twitter (X), and using the Scalekit SDK to add "Sign in with Twitter" to your application.

<figure>
  ![A diagram showing "Your Application" connecting to "Scalekit" via OpenID Connect, which links to Twitter (X) using OAuth 2.0.](@/assets/docs/common/poster-scalekit-social.png)
</figure>

By the end of this guide, you will be able to:

1. Set up an OAuth 2.0 connection between Scalekit and Twitter (X)
2. Implement "Sign in with Twitter" in your application using the Scalekit SDK

## Set up Twitter (X) connection

### Access social login settings

Open your Scalekit dashboard and navigate to **Social Login** under the **Authentication** section.

<figure>
  ![Scalekit dashboard showcasing social login setup with various platform integration options.](@/assets/docs/common/social-connections/1-navigate-to-social-logins.png)
</figure>

### Add Twitter (X) connection

Click the **+ Add Connection** button and select **Twitter (X)** from the list of available options.

<figure>
  ![Add social login connections: Google, Microsoft, GitHub, Twitter (X).](@/assets/docs/common/social-connections/2-list-social-logins.png)
  <figcaption>Add social login connections: Twitter (X)</figcaption>
</figure>

## Configure OAuth settings

The OAuth Configuration details page helps you set up the connection:

- Note the **Redirect URI** provided for your app. You'll use this URL to register with Twitter (X).
- **Client ID** and **Client Secret** are generated by Twitter (X) when you register an OAuth App. They enable Scalekit to authenticate your app and establish trust with Twitter.

<figure>
  ![Twitter (X) OAuth configuration in Scalekit, showing redirect URI, client credentials, and scopes for social login setup.](@/assets/docs/common/social-connections/3-configure-oauth-settings.png)
</figure>

### Set up Twitter (X) OAuth 2.0

Twitter (X) uses OAuth 2.0 with PKCE (Proof Key for Code Exchange). <a href="https://developer.twitter.com/en/docs/authentication/oauth-2-0/authorization-code" target="_blank" rel="noopener">Follow Twitter's instructions to set up OAuth 2.0</a>.

<Steps>
1. Go to the <a href="https://developer.twitter.com/en/portal/dashboard" target="_blank" rel="noopener">Twitter Developer Portal</a> and sign in with your Twitter account.

2. Create a new project and app (or use an existing one):
   - Click **+ Create Project** and follow the setup wizard
   - Give your project and app meaningful names

3. In your app settings, navigate to **User authentication settings** and click **Set up**:
   - **App permissions**: Select **Read** (or broader if your app requires it)
   - **Type of App**: Select **Web App, Automated App or Bot**
   - **Callback URI / Redirect URL**: Paste the **Redirect URI** from Scalekit
   - **Website URL**: Enter your application's homepage URL

4. Click **Save** to apply the authentication settings.

5. Navigate to **Keys and tokens** in your app's dashboard:
   - Under **OAuth 2.0 Client ID and Client Secret**, click **Generate** (or **Regenerate** if credentials already exist)
   - Copy the **Client ID** and **Client Secret**

6. Paste the **Client ID** and **Client Secret** into the Scalekit dashboard.

7. Click **Save Changes** in Scalekit to complete the setup.
</Steps>

## Test the connection

Click the **Test Connection** button in Scalekit. You will be redirected to the Twitter (X) authorization screen to grant access. A summary table will show the information that will be sent to your app.
