## Getting resource IDs

Many tools require IDs that you must fetch first:

| Resource | Tool to get ID | Field in response |
|----------|---------------|-------------------|
| Account ID | `jira_myself_get` | `accountId` |
| Project key | `jira_projects_list` | `values[].key` (e.g. `KAN`) |
| Issue key | `jira_issues_search` (JQL: `project = KAN`) | `issues[].key` (e.g. `KAN-1`) |
| Issue ID | `jira_issues_search` | `issues[].id` |
| Comment ID | `jira_issue_comments_list` | `comments[].id` |
| Worklog ID | `jira_issue_worklogs_list` | `worklogs[].id` |
| Attachment ID | `jira_issue_get` (expand: `attachments`) | `fields.attachment[].id` |
| Transition ID | `jira_issue_transitions_list` | `transitions[].id` |
| Component ID | `jira_project_components_list` | `values[].id` |
| Version ID | `jira_project_versions_list` | `values[].id` |
| Issue link ID | `jira_issue_get` (fields: `issuelinks`) | `fields.issuelinks[].id` |
| Remote link ID | `jira_issue_remote_links_list` | `[].id` |
| Filter ID | `jira_filters_search` | `values[].id` |
| Issue type ID | `jira_issue_types_list` | `[].id` |
| Priority ID | `jira_priorities_list` | `[].id` |
| Role ID | `jira_roles_list` | `[].id` |
| Permission scheme ID | `jira_permission_schemes_list` | `permissionSchemes[].id` |
| Notification scheme ID | `jira_notification_schemes_list` | `values[].id` |
| Group name | `jira_groups_find` | `groups[].name` |

## Tool list

The Jira connector provides 96 tools covering the full Jira API — issues, projects, components, versions, comments, worklogs, filters, transitions, remote links, JQL utilities, and administration.


## `jira_attachment_delete`

Permanently delete a Jira issue attachment by its ID. This action cannot be undone. Requires Delete Attachments project permission.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `id` | string | Yes | Jira attachment ID to permanently delete |


## `jira_attachment_get`

Get metadata for a Jira issue attachment by its ID. Returns the filename, MIME type, size, creation date, author, and download URL.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `id` | string | Yes | Jira attachment ID |


## `jira_component_create`

Create a new component in a Jira project. Components are used to group and categorize issues within a project.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `assigneeType` | string | No | Who issues in this component are assigned to by default |
| `description` | string | No | Brief description of what this component covers |
| `leadAccountId` | string | No | Atlassian account ID of the component lead |
| `name` | string | Yes | Display name for the component |
| `project` | string | Yes | Project key where this component will be created |


## `jira_component_delete`

Delete a Jira project component by its ID. Optionally move issues from the deleted component to another component. Requires Administer Projects permission.

| Name | Type | Required | Description |
| --- | --- | --- | --- |

| `moveIssuesTo` | string | No | If provided, issues in this component will be moved to this component ID |
