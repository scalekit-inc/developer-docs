## Tool list

## `jira_attachment_delete`

Permanently delete a Jira issue attachment by its ID. This action cannot be undone. Requires Delete Attachments project permission.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `id` | string | Yes | The attachment ID to delete |

## `jira_attachment_get`

Get metadata for a Jira issue attachment by its ID. Returns the filename, MIME type, size, creation date, author, and download URL.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `id` | string | Yes | The attachment ID to retrieve metadata for |

## `jira_component_create`

Create a new component in a Jira project. Components are used to group and categorize issues within a project.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `assigneeType` | string | No | Default assignee type: PROJECT_DEFAULT, COMPONENT_LEAD, PROJECT_LEAD, or UNASSIGNED |
| `description` | string | No | Description of the component |
| `leadAccountId` | string | No | Account ID of the component lead |
| `name` | string | Yes | Name of the component |
| `project` | string | Yes | Key of the project to add the component to |

## `jira_component_delete`

Delete a Jira project component by its ID. Optionally move issues from the deleted component to another component. Requires Administer Projects permission.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `id` | string | Yes | The component ID to delete |
| `moveIssuesTo` | string | No | Component ID to move issues to after deleting this component |

## `jira_component_get`

Retrieve details of a Jira project component by its ID, including name, description, lead, and default assignee settings.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `id` | string | Yes | The component ID to retrieve |

## `jira_component_update`

Update an existing Jira project component's name, description, lead, or default assignee settings.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `assigneeType` | string | No | Updated default assignee type: PROJECT_DEFAULT, COMPONENT_LEAD, PROJECT_LEAD, or UNASSIGNED |
| `description` | string | No | Updated component description |
| `id` | string | Yes | The component ID to update |
| `leadAccountId` | string | No | Account ID of the new component lead |
| `name` | string | No | Updated component name |

## `jira_field_search`

Search for Jira fields by name, type, or other criteria with pagination support. Returns paginated field results.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `maxResults` | integer | No | Maximum number of fields to return (default 50) |
| `orderBy` | string | No | Sort by: contextsCount, lastUsed, name, screensCount, or -prefixed for descending |
| `query` | string | No | Search query to filter fields by name (case-insensitive) |
| `startAt` | integer | No | Index of the first field to return (default 0) |
| `type` | string | No | Filter by field type: custom or system |

## `jira_fields_list`

Get all system and custom fields available in Jira. Returns field IDs, names, types, and whether they are custom or system fields. Use field IDs when referencing fields in JQL or issue creation.

## `jira_filter_create`

Create a saved Jira filter with a JQL query. Filters can be shared, added to favorites, and used on Jira dashboards.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `description` | string | No | Description of what this filter shows |
| `favourite` | boolean | No | Whether to add this filter to favorites immediately |
| `jql` | string | No | JQL query string for this filter |
| `name` | string | Yes | Name of the filter (must be unique for the user) |

## `jira_filter_delete`

Permanently delete a saved Jira filter. Only the filter owner or admins can delete a filter. This action cannot be undone.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `id` | string | Yes | The filter ID to delete |

## `jira_filter_get`

Retrieve a saved Jira filter by its ID, including the JQL query, name, owner, and share permissions.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `expand` | string | No | Additional data to include (e.g. sharedUsers, subscriptions) |
| `id` | string | Yes | The filter ID to retrieve |

## `jira_filter_update`

Update a saved Jira filter's name, description, or JQL query. Only the filter owner or admins can update a filter.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `description` | string | No | Updated description of the filter |
| `id` | string | Yes | The filter ID to update |
| `jql` | string | No | Updated JQL query string |
| `name` | string | Yes | Updated filter name |

## `jira_filters_search`

Search for saved Jira filters with pagination. Filter results by name, owner, project, or group. Returns filter details including JQL queries.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `accountId` | string | No | Filter by filter owner account ID |
| `expand` | string | No | Additional data to include (e.g. description, favourite, sharePermissions) |
| `filterName` | string | No | Search by filter name (partial match, case-insensitive) |
| `maxResults` | integer | No | Maximum number of filters to return (default 50) |
| `orderBy` | string | No | Field to order by (e.g. name, id, owner, favourite_count, is_favourite) |
| `startAt` | integer | No | Index of the first filter to return (default 0) |

## `jira_group_member_add`

Add a user to a Jira group. Requires Administer Jira global permission or the Site Administration role.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `accountId` | string | Yes | Account ID of the user to add to the group |
| `groupId` | string | No | The group ID to add the user to (use instead of groupname) |
| `groupname` | string | No | The group name to add the user to |

## `jira_group_member_remove`

Remove a user from a Jira group by their account ID. Requires Administer Jira global permission.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `accountId` | string | Yes | Account ID of the user to remove from the group |
| `groupId` | string | No | The group ID to remove the user from (use instead of groupname) |
| `groupname` | string | No | The group name to remove the user from |

## `jira_group_members_list`

Get a paginated list of users in a Jira group. Returns account IDs, display names, and email addresses of group members.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `groupId` | string | No | The group ID to list members of (use instead of groupname) |
| `groupname` | string | No | The group name to list members of |
| `includeInactiveUsers` | boolean | No | Whether to include inactive (deactivated) users in the results |
| `maxResults` | integer | No | Maximum number of members to return (default 50) |
| `startAt` | integer | No | Index of the first member to return (default 0) |

## `jira_groups_find`

Find Jira user groups by name. Returns groups whose names match the query. Useful for finding group names to use in permission schemes or visibility restrictions.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `accountId` | string | No | Filter to only return groups the user with this account ID belongs to |
| `excludeId` | string | No | Group IDs to exclude from results (comma-separated) |
| `maxResults` | integer | No | Maximum number of groups to return (default 20) |
| `query` | string | No | Search string to match against group names |

## `jira_issue_assign`

Assign or unassign a Jira issue to a user. Pass an accountId to assign, or omit/null to unassign. The user must have the Assign Issues project permission.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `accountId` | string | No | Account ID of the user to assign. Leave null or omit to unassign. |
| `issueIdOrKey` | string | Yes | The issue ID or key to assign (e.g. PROJ-123) |

## `jira_issue_changelog_list`

Get the paginated change history for a Jira issue. Returns a list of changelog entries showing which fields changed, who changed them, and when.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `issueIdOrKey` | string | Yes | The issue ID or key to retrieve changelog for |
| `maxResults` | integer | No | Maximum number of changelog entries to return (default 100) |
| `startAt` | integer | No | Index of the first entry to return for pagination (default 0) |

## `jira_issue_comment_add`

Add a comment to a Jira issue. The comment body is plain text and will be wrapped in ADF (Atlassian Document Format) for the v3 API. Optionally restrict visibility to a specific role or group.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `body` | string | Yes | The plain-text content of the comment |
| `issueIdOrKey` | string | Yes | The issue ID or key to add the comment to |
| `visibility_type` | string | No | Restrict comment visibility by type: 'role' or 'group' |
| `visibility_value` | string | No | Name of the role or group to restrict visibility to |

## `jira_issue_comment_delete`

Permanently delete a comment from a Jira issue. Only the comment author or users with Administer Projects permission can delete comments. This action cannot be undone.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `id` | string | Yes | The comment ID to delete |
| `issueIdOrKey` | string | Yes | The issue ID or key the comment belongs to |

## `jira_issue_comment_get`

Retrieve a specific comment on a Jira issue by comment ID. Returns the comment body, author, and timestamps.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `expand` | string | No | Additional fields to include (e.g. renderedBody for HTML content) |
| `id` | string | Yes | The comment ID to retrieve |
| `issueIdOrKey` | string | Yes | The issue ID or key the comment belongs to |

## `jira_issue_comment_update`

Update the body of an existing comment on a Jira issue. Only the comment author or users with Administer Projects permission can update comments.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `body` | string | Yes | The new plain-text content for the comment |
| `id` | string | Yes | The comment ID to update |
| `issueIdOrKey` | string | Yes | The issue ID or key the comment belongs to |
| `notifyUsers` | boolean | No | Whether to send notifications to watchers (default true) |

## `jira_issue_comments_list`

Get all comments for a Jira issue with pagination support. Returns comment bodies, author details, and timestamps. Use expand=renderedBody to get HTML-rendered comment content.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `expand` | string | No | Additional fields to include (e.g. renderedBody for HTML content) |
| `issueIdOrKey` | string | Yes | The issue ID or key to list comments for |
| `maxResults` | integer | No | Maximum number of comments to return (default 50) |
| `orderBy` | string | No | Field to order by (created or -created for descending) |
| `startAt` | integer | No | Index of the first comment to return (default 0) |

## `jira_issue_create`

Create a new Jira issue or subtask in a specified project. Requires a project key, issue type, and summary. Supports assigning users, setting priority, labels, components, parent issue (for subtasks), and a plain-text description.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `assignee_account_id` | string | No | Account ID of the user to assign this issue to |
| `components` | `array<string>` | No | List of component names to associate with this issue |
| `description` | string | No | Plain-text description of the issue (wrapped in ADF for v3 API) |
| `fix_versions` | `array<string>` | No | List of version names to set as fix versions |
| `issue_type` | string | Yes | Name of the issue type (e.g. Bug, Story, Task, Sub-task) |
| `labels` | `array<string>` | No | List of labels to apply to the issue |
| `parent_key` | string | No | Key of the parent issue (required for Sub-task issue type) |
| `priority_name` | string | No | Priority name for the issue (e.g. Highest, High, Medium, Low, Lowest) |
| `project_key` | string | Yes | Key of the project to create the issue in (e.g. PROJ) |
| `summary` | string | Yes | Short summary or title of the issue |

## `jira_issue_delete`

Permanently delete a Jira issue and all its subtasks (if deleteSubtasks is true). This action cannot be undone. The user must have permission to delete the issue.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `deleteSubtasks` | string | No | Whether to delete subtasks of this issue (required if the issue has subtasks) |
| `issueIdOrKey` | string | Yes | The issue ID or key to delete (e.g. PROJ-123) |

## `jira_issue_get`

Retrieve details of a Jira issue by its ID or key. Returns fields, status, assignee, priority, comments summary, and other metadata. Use the fields parameter to limit the response to specific fields.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `expand` | string | No | Comma-separated list of additional data to include (e.g. renderedFields,names,changelog) |
| `fields` | string | No | Comma-separated list of fields to return (use * for all, -field to exclude) |
| `issueIdOrKey` | string | Yes | The issue ID (e.g. 10001) or key (e.g. PROJ-123) to retrieve |
| `properties` | string | No | Comma-separated list of issue properties to return |
| `updateHistory` | boolean | No | Whether to update the issue's viewed history for the current user |

## `jira_issue_worklog_add`

| `timeSpent` | string | Yes | Time spent in Jira duration format (e.g. '2h 30m', '1d', '45m') |
