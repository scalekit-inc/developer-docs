Connect to Linear. Manage issues, projects, sprints, and development workflows

Supports authentication: <Badge text="OAuth 2.0" />


## `linear_comment_create`

Create a comment on an issue in Linear using the commentCreate mutation.

| Properties | Description | Type |
| --- | --- | --- |
| `body` | The comment text | string |
| `issue_id` | ID of the issue to comment on | string |

## `linear_comment_delete`

Archive or delete a comment in Linear using the commentDelete mutation. This action is destructive and cannot be undone.

| Properties | Description | Type |
| --- | --- | --- |
| `comment_id` | ID of the comment to delete | string |

## `linear_comment_read`

Retrieve details of a specific comment by ID using the comment query.

| Properties | Description | Type |
| --- | --- | --- |
| `comment_id` | ID of the comment to retrieve | string |

## `linear_comment_update`

Update an existing comment in Linear using the commentUpdate mutation.

| Properties | Description | Type |
| --- | --- | --- |
| `body` | New comment text | string |
| `comment_id` | ID of the comment to update | string |

## `linear_comments_list`

List comments in Linear using the comments query with filtering and pagination support.

| Properties | Description | Type |
| --- | --- | --- |
| `after` | Cursor for pagination (returns comments after this cursor) | string | null |
| `first` | Number of comments to return (pagination) | number | null |
| `issue_id` | Filter comments by issue ID | string | null |
| `user_id` | Filter comments by user ID | string | null |

## `linear_issue_create`

Create a new issue in Linear using the issueCreate mutation. Requires a team ID and title at minimum.

| Properties | Description | Type |
| --- | --- | --- |
| `assignee_id` | ID of the user to assign the issue to | string | null |
| `description` | Description of the issue | string | null |
| `estimate` | Story point estimate for the issue | number | null |
| `label_ids` | Array of label IDs to apply to the issue | `array<string>` | null |
| `priority` | Priority level of the issue (1-4, where 1 is urgent) | number | null |
| `state_id` | ID of the workflow state to set | string | null |
| `team_id` | ID of the team to create the issue in | string |
| `title` | Title of the issue | string |
