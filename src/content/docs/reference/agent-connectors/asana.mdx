      // Using a self-executing function to avoid global namespace pollution
      (function() {
        // Function to initialize the copy functionality
        function initHeadingCopy() {
          const headings = document.querySelectorAll('.sl-markdown-content h2, .sl-markdown-content h3, .sl-markdown-content h4');

          headings.forEach(heading => {
            // Skip if already initialized
            if (heading.hasAttribute('data-copy-initialized')) return;
            heading.setAttribute('data-copy-initialized', 'true');

            heading.addEventListener('click', async () => {
              const headingText = heading.textContent.replace(/\s📋$/, '').trim();

              try {
                await navigator.clipboard.writeText(headingText);

                // Visual feedback
                const originalText = heading.textContent;
                heading.textContent = heading.textContent.replace(/\s📋$/, '') + ' ✓';

                setTimeout(() => {
                  heading.textContent = originalText;
                }, 2000);
              } catch (err) {
                console.error('Failed to copy heading:', err);
              }
            });
          });
        }

        // Initialize on DOMContentLoaded
        document.addEventListener('DOMContentLoaded', initHeadingCopy);

        // Also initialize now in case the DOM is already loaded
        if (document.readyState === 'complete' || document.readyState === 'interactive') {
          setTimeout(initHeadingCopy, 1);
        }

        // For Astro's client-side navigation
        document.addEventListener('astro:page-load', initHeadingCopy);
      })();

## Setup

<Aside type="caution">
  Asana OAuth applications that access user data may require additional verification steps through Asana's app review process before going to production.
</Aside>

### Configure Redirect URL

<Steps>
1. In [Scalekit dashboard](https://app.scalekit.com), go to **Agent Auth** → **Create Connection**.

   {/* Screenshot: Agent Auth → Create Connection */}

2. Find **Asana** from the list of providers and click **Create**.

   <Aside>
     By default, a connection using Scalekit's credentials will be created. If you are testing, go directly to [Usage](#usage). Before going to production, update your connection by following the steps below.
   </Aside>

3. Click **Use your own credentials** and copy the redirect URI — it will look like `https://{your-env-url}/sso/v1/oauth/{connection_id}/callback`.

   {/* Screenshot: Use your own credentials panel with redirect URI */}

4. Navigate to the [Asana Developer Console](https://app.asana.com/0/developer-console) and select **Create new app**.

5. Enter a name and a description for your application, then click **Create app**.

   {/* Screenshot: Asana Create new app button */}

6. In your app settings, go to **OAuth** and under **Redirect URLs**, click **Add redirect URL**.

7. Paste the redirect URI copied from Scalekit and click **Add**.

   {/* Screenshot: Asana redirect URL field */}
</Steps>

Asana provides your Client ID and Client Secret in the **OAuth** section of your app settings.

### Add Credentials

<Steps>
1. In [Scalekit dashboard](https://app.scalekit.com), go to **Agent Auth** → **Connections** and open the connection you created.

2. Enter your credentials:
   - Client ID (from above)
   - Client Secret (from above)
   - Permissions (scopes — see [Asana OAuth Scopes](https://developers.asana.com/docs/oauth#scopes))

   {/* Screenshot: Add credentials form */}

3. Click **Save**
</Steps>
