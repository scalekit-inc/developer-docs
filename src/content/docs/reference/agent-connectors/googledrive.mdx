      // Using a self-executing function to avoid global namespace pollution
      (function() {
        // Function to initialize the copy functionality
        function initHeadingCopy() {
          const headings = document.querySelectorAll('.sl-markdown-content h2, .sl-markdown-content h3, .sl-markdown-content h4');

          headings.forEach(heading => {
            // Skip if already initialized
            if (heading.hasAttribute('data-copy-initialized')) return;
            heading.setAttribute('data-copy-initialized', 'true');

            heading.addEventListener('click', async () => {
              const headingText = heading.textContent.replace(/\s📋$/, '').trim();

              try {
                await navigator.clipboard.writeText(headingText);

                // Visual feedback
                const originalText = heading.textContent;
                heading.textContent = heading.textContent.replace(/\s📋$/, '') + ' ✓';

                setTimeout(() => {
                  heading.textContent = originalText;
                }, 2000);
              } catch (err) {
                console.error('Failed to copy heading:', err);
              }
            });
          });
        }

        // Initialize on DOMContentLoaded
        document.addEventListener('DOMContentLoaded', initHeadingCopy);

        // Also initialize now in case the DOM is already loaded
        if (document.readyState === 'complete' || document.readyState === 'interactive') {
          setTimeout(initHeadingCopy, 1);
        }

        // For Astro's client-side navigation
        document.addEventListener('astro:page-load', initHeadingCopy);
      })();

## Setup

<Aside type="caution">
  Google applications using scopes that permit access to certain user data must complete a verification process.
</Aside>

### Configure redirect URL

<Steps>
1. ## Open the Scalekit dashboard

   In [Scalekit dashboard](https://app.scalekit.com), go to **Agent Auth** → **Create Connection**.

2. ## Create a Google Drive connection

   Find **Google Drive** from the list of providers and click **Create**.

3. ## Copy the redirect URI

   Click **Use your own credentials** and copy the redirect URI — it will look like `https://{your-env-url}/sso/v1/oauth/{connection_id}/callback`.

   <img src={useOwnCredentialsImg.src} alt="Configure Google Drive Connection with redirect URI" width="500" />

4. ## Open Google Cloud Console

   Navigate to [Google Cloud Console's](https://console.cloud.google.com/projectselector2/home/dashboard?supportedpurview=project) **APIs & Services > Credentials**

5. ## Create an OAuth client ID

   Select **+ Create Credentials**, then **OAuth client ID**

6. ## Select web application type

   Choose **Web application** from the Application type menu

   <img src={oauthWebAppImg.src} alt="Selecting Web Application in Google OAuth" width="540" />

7. ## Add the redirect URI

   Under **Authorized redirect URIs**, click **+ Add URI**, paste the redirect URI, and select **Create**.

   <img src={addRedirectUriImg.src} alt="Adding Google redirect URI for OAuth" width="540" />
</Steps>

Google provides your Client ID and Client Secret credentials.

### Enable Google Drive API

<Steps>
1. ## Open the API Library

   In [Google Cloud Console](https://console.cloud.google.com/projectselector2/home/dashboard?supportedpurview=project), go to **APIs & Services > Library**

2. ## Search for the Google Drive API

   Search for "Google Drive" in the API Library

3. ## Select the Google Drive API

   Select **Google Drive API**

4. ## Enable the API

   Press the blue **Enable** button
</Steps>

### Add credentials

<Steps>
1. ## Open your connection settings

   In [Scalekit dashboard](https://app.scalekit.com), go to **Agent Auth** → **Connections** and open the connection you created.

2. ## Enter your credentials

   Enter your credentials:
   - Client ID (from above)
   - Client Secret (from above)
   - Permissions (scopes — see [Google API Scopes reference](https://developers.google.com/identity/protocols/oauth2/scopes))

   <img src={addCredentialsImg.src} alt="Add credentials" width="540" />

3. ## Save the connection

   Click **Save**
</Steps>

## `googledrive_get_file_metadata`

Retrieve metadata for a specific file in Google Drive by its file ID. Returns name, MIME type, size, creation time, and more.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `fields` | string | No | Fields to include in the response |
| `file_id` | string | Yes | The ID of the file to retrieve metadata for |
| `schema_version` | string | No | Optional schema version to use for tool execution |
| `supports_all_drives` | boolean | No | Support shared drives |
| `tool_version` | string | No | Optional tool version to use for execution |

## `googledrive_search_content`

Search inside the content of files stored in Google Drive using full-text search. Finds files where the body text matches the search term.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `fields` | string | No | Fields to include in the response |
| `mime_type` | string | No | Filter results by MIME type |
| `page_size` | integer | No | Number of files to return per page |
| `page_token` | string | No | Token for the next page of results |
| `schema_version` | string | No | Optional schema version to use for tool execution |
| `search_term` | string | Yes | Text to search for inside file contents |
| `supports_all_drives` | boolean | No | Include shared drives in results |
| `tool_version` | string | No | Optional tool version to use for execution |

## `googledrive_search_files`

Search for files and folders in Google Drive using query filters like name, type, owner, and parent folder.

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `fields` | string | No | Fields to include in the response |
| `order_by` | string | No | Sort order for results |
| `page_size` | integer | No | Number of files to return per page |
| `page_token` | string | No | Token for the next page of results |
| `query` | string | No | Drive search query string |
| `schema_version` | string | No | Optional schema version to use for tool execution |
| `supports_all_drives` | boolean | No | Include shared drives in results |
| `tool_version` | string | No | Optional tool version to use for execution |
