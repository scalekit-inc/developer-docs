## Organization events

### `organization.created`

This webhook is triggered when a new organization is created. The event type is `organization.created`

```json title="organization.created"
{
  "environment_id": "env_1234567890",
  "id": "evt_1234567890",
  "object": "Organization",
  "occurred_at": "2024-01-15T10:30:00.123456789Z",
  "organization_id": "org_1234567890",
  "spec_version": "1",
  "type": "organization.created",
  "data": {
    "create_time": "2025-12-09T09:25:02.02Z",
    "display_name": "AcmeCorp",
    "external_id": "org_external_123",
    "id": "org_1234567890",
    "metadata": null,
    "region_code": "US",
    "update_time": "2025-12-09T09:25:02.025330364Z",
    "settings": {
      "features": [
        {
          "enabled": true,
          "name": "sso"
        },
        {
          "enabled": false,
          "name": "dir_sync"
        },
        {
          "enabled": false,
          "name": "domain_verification"
        }
      ]
    }
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier for the organization |
| `external_id` | string \| null | External identifier for the organization, if provided |
| `display_name` | string \| null | Name of the organization, if provided |
| `region_code` | string \| null | Geographic region code for the organization (US, EU), currently limited to US |
| `create_time` | string | Timestamp of when the organization was created  |
| `update_time` | string \| null | Timestamp of when the organization was last updated |
| `metadata` | object \| null | Additional metadata associated with the organization |
| `settings` | object \| null | Organization settings including feature flags (sso, dir_sync, domain_verification, session_policy) |
| `settings.features` | array | Array of feature objects with enabled status and name |

## Organization domain events

These events fire whenever a domain is added, verified, or removed against an organization. They share a common payload shape derived from the Domain object. See [Organization domains](/authenticate/manage-users-orgs/organization-domains/) for the underlying workflow.

### `organization.domain_created`

This webhook is triggered when a domain is added to an organization. The event type is `organization.domain_created`.

```json title="organization.domain_created"
{
  "spec_version": "1",
  "id": "evt_94567862441607493",
  "object": "Domain",
  "environment_id": "env_74418471961625391",
  "occurred_at": "2025-10-14T09:27:18.488720586Z",
  "organization_id": "org_83544995172188677",
  "type": "organization.domain_created",
  "data": {
    "id": "dom_9a8b7c6d5e4f",
    "domain": "zapcorp.com",
    "domain_type": "ORGANIZATION_DOMAIN",
    "organization_id": "org_83544995172188677",
    "environment_id": "env_74418471961625391",
    "txt_record_key": "_scalekit-verify",
    "txt_record_secret": "scalekit-verification=sk_dvr_8f2c4a1e9b6d7e3f",
    "verification_status": "PENDING",
    "verification_method": "DNS",
    "updated_at": "2025-10-14T09:27:18.488720586Z"
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier for the domain |
| `domain` | string | The registered domain name (lowercase) |
| `domain_type` | string | `ORGANIZATION_DOMAIN` or `ALLOWED_EMAIL_DOMAIN` |
| `organization_id` | string | Identifier of the organization that owns the domain |
| `environment_id` | string | Identifier of the Scalekit environment |
| `txt_record_key` | string | DNS record name used for DNS verification |
| `txt_record_secret` | string | DNS record value used for DNS verification |
| `verification_status` | string | `PENDING`, `VERIFIED`, or `FAILED` |
| `verification_method` | string | `ADMIN`, `DNS`, or `NOT_APPLICABLE` |
| `updated_at` | string | Timestamp of the most recent change to the domain |

### `organization.domain_dns_verification_failed`

This webhook is triggered when DNS verification expires before Scalekit can find the matching TXT record, and the domain transitions to `verification_status: FAILED`. The event type is `organization.domain_dns_verification_failed`.

```json title="organization.domain_deleted"
  "id": "evt_94568318877245912",
  "object": "Domain",
  "environment_id": "env_74418471961625391",
  "occurred_at": "2025-10-16T11:14:09.337417Z",
  "organization_id": "org_83544995172188677",
  "type": "organization.domain_deleted",
  "data": {
    "id": "dom_9a8b7c6d5e4f",
    "domain": "zapcorp.com",
    "domain_type": "ORGANIZATION_DOMAIN",
    "organization_id": "org_83544995172188677",
    "environment_id": "env_74418471961625391",
    "txt_record_key": "_scalekit-verify",
    "txt_record_secret": "scalekit-verification=sk_dvr_8f2c4a1e9b6d7e3f",
    "verification_status": "VERIFIED",
    "verification_method": "DNS",
    "updated_at": "2025-10-16T11:14:09.337417Z"
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier for the domain |
| `domain` | string | The registered domain name |
| `domain_type` | string | `ORGANIZATION_DOMAIN` or `ALLOWED_EMAIL_DOMAIN` |
| `organization_id` | string | Identifier of the organization the domain belonged to |
| `environment_id` | string | Identifier of the Scalekit environment |
| `txt_record_key` | string | DNS record name that was issued for verification |
| `txt_record_secret` | string | DNS record value that was issued for verification |
| `verification_status` | string | Last known verification status before deletion |
| `verification_method` | string | Method recorded at the time of deletion |
| `updated_at` | string | Timestamp of the deletion |
