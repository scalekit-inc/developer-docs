The **Scalekit Agent Auth** skill for [OpenClaw](https://github.com/scalekit-inc/openclaw-skill) lets your OpenClaw agent execute actions on third-party services directly from your conversation — searching LinkedIn, reading Notion pages, sending Slack messages, querying Snowflake, and more — all through Scalekit Connect.

## How it works

When you ask Claude to interact with a third-party service, the skill:

1. Discovers the configured connection for the provider in Scalekit (e.g. [Gmail connection setup](/reference/agent-connectors/gmail/))
2. Verifies the connection is active (OAuth magic link for new OAuth connections; Dashboard guidance for API Key / Bearer / Basic connections)
3. Fetches the available tools for that provider and their schemas
4. Executes the right tool with the correct parameters
5. Falls back to a proxied HTTP request if no tool exists for the action

<Aside type="tip">
  The skill automatically detects whether a connection is OAuth or non-OAuth and applies the correct auth flow — no configuration needed.
</Aside>

Your agent never stores tokens or API keys. Scalekit acts as a token vault — all OAuth tokens, API keys, and credentials are stored and managed by Scalekit. The skill retrieves only what it needs at runtime, scoped to the requesting user.

## Prerequisites

- [OpenClaw](https://openclaw.ai) installed and set up
- A Scalekit account with Agent Auth enabled — [get started at app.scalekit.com](https://app.scalekit.com)
- `python3` and `uv` available in your PATH

## Installation

Install the skill from ClawHub:

```bash
clawhub install scalekit-agent-auth
```

Then add your Scalekit credentials to `.env` in your project root:

```bash
TOOL_CLIENT_ID=skc_your_client_id
TOOL_CLIENT_SECRET=your_client_secret
TOOL_ENV_URL=https://your-env.scalekit.cloud
TOOL_IDENTIFIER=your_default_user_identifier
```

<Aside type="caution">
  `TOOL_CLIENT_ID`, `TOOL_CLIENT_SECRET`, and `TOOL_ENV_URL` are required. `TOOL_IDENTIFIER` is optional but recommended — it sets the default user context for all tool calls.
</Aside>

## Example: Read emails with Gmail

```
You: Show me my latest unread emails
```

OpenClaw will automatically:
1. Look up the `GMAIL` connection
2. Verify it's active (or generate a magic link to authorize if needed)
3. Fetch the `gmail_list_emails` tool schema
4. Return your latest unread emails

## Example: Read a Notion page

```
You: Read my Notion page https://notion.so/My-Page-abc123
```

OpenClaw will:
1. Look up the `NOTION` connection
2. If not yet authorized, generate a magic link for you to complete OAuth
3. Fetch the `notion_page_get` tool schema
4. Return the page content

## Supported providers

Any provider configured in Scalekit — including Notion, Slack, Gmail, Google Sheets, GitHub, Salesforce, HubSpot, Linear, Snowflake, Exa, HarvestAPI, and 50+ more.
