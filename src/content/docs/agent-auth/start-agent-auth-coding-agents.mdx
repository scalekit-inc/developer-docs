Use AI coding agents to accelerate your Agent Auth integration. This guide shows you how to configure and use coding agents like Claude Code to implement Scalekit's Agent Auth end-to-end in your AI applications.

AI-assisted development reduces implementation time from hours to minutes. The coding agent analyzes your codebase, applies authentication patterns, and generates production-ready code that follows security best practices for connecting AI agents to external services like Gmail, Calendar, Slack, and Notion.

**Prerequisites**

- A [Scalekit account](https://app.scalekit.com) with Agent Auth access
- Basic familiarity with OAuth 2.0 and AI agent architectures
- Terminal access for installing coding agent tools
- At least one active connection in your Scalekit dashboard (Gmail, Google Calendar, Slack, etc.)

<Steps>

1. ## Install your coding agent

   Install an AI coding agent that supports plugin ecosystems. Claude Code provides native integration with Scalekit's authentication plugins.

   <Tabs>
   <TabItem value="claude" label="Claude Code">

   Run the installation script to set up Claude Code on your system:

   ```bash title="Terminal" frame="terminal" showLineNumbers=false
   curl -fsSL https://claude.ai/install.sh | bash
   ```

   After installation completes, verify setup by running:

   ```bash title="Terminal" frame="terminal" showLineNumbers=false
   claude --version
   ```

   Learn more about Claude Code in the [official quickstart guide](https://code.claude.com/docs/en/quickstart).

   </TabItem>
   </Tabs>

   <Aside type="tip" title="Other AI coding tools">
   While this guide focuses on Claude Code, same authentication patterns apply to other AI-assisted development tools. The implementation principles remain consistent regardless of your chosen coding agent.
   </Aside>

2. ## Add the Scalekit Auth Stack marketplace

   Register Scalekit's plugin marketplace to access pre-configured authentication skills. This marketplace provides context-aware prompts and implementation guides that help coding agents generate correct Agent Auth code.

   <Tabs>
   <TabItem value="claude" label="Claude Code">

   Start the Claude Code REPL and add the marketplace:

   ```bash title="Terminal" frame="terminal" showLineNumbers=false
   # Start Claude REPL
   claude

   # Add Scalekit Auth Stack marketplace
   /plugin marketplace add scalekit-inc/claude-code-authstack
   ```

   When the marketplace registers successfully, you'll see confirmation output:

   ```bash title="Terminal" frame="terminal" showLineNumbers=false
   ❯ /plugin marketplace add scalekit-inc/claude-code-authstack
      ⎿  Successfully added marketplace: scalekit-auth-stack
   ```

   </TabItem>
   </Tabs>

   The marketplace provides specialized authentication plugins that understand Agent Auth patterns and OAuth 2.0 security requirements. These plugins guide the coding agent to generate implementation code that matches your project structure.

   To remove the Scalekit Auth Stack marketplace, use the uninstall command:

   <Tabs>
   <TabItem value="claude" label="Claude Code">

   ```bash title="Claude REPL" showLineNumbers=false
   /plugin marketplace remove scalekit-auth-stack
   ```

   </TabItem>
   </Tabs>

3. ## Enable authentication plugins

   Select which authentication capabilities to activate in your development environment. Each plugin provides specific skills that the coding agent uses to generate authentication code.

   Run the plugin wizard to browse and enable available plugins:

   <Tabs>
   <TabItem value="claude" label="Claude Code">

   ```bash title="Claude REPL" showLineNumbers=false
   /plugins
   ```

   </TabItem>
   </Tabs>

   Navigate through the visual interface to enable the Agent Auth plugin.

   <Aside type="tip" title="Auto-update recommendations">
   Enable auto-updates for authentication plugins to receive security patches and improvements. Scalekit regularly updates plugins based on community feedback and security best practices.
   </Aside>

   Or, directly install the specific plugin:

   <Tabs>
   <TabItem value="claude" label="Claude Code">

   ```bash title="Claude REPL" showLineNumbers=false
   /plugin install agent-auth@scalekit-auth-stack
   ```

   </TabItem>
   </Tabs>

   To remove an installed plugin, use the uninstall command:

   <Tabs>
   <TabItem value="claude" label="Claude Code">

   ```bash title="Claude REPL" showLineNumbers=false
   /plugin uninstall agent-auth@scalekit-auth-stack
   ```

   </TabItem>
   </Tabs>

4. ## Generate authentication implementation

   Use a structured prompt to direct the coding agent. A well-formed prompt ensures the agent generates complete, production-ready Agent Auth code that includes all required security components.

   Copy the following prompt into your coding agent:

   ```md wrap showLineNumbers=false title="Authentication implementation prompt"
   Add Agent Auth to my AI application using Scalekit. Initialize ScalekitClient with environment credentials, implement functions to create connected accounts for users, generate authorization links for OAuth consent, and fetch OAuth tokens for making authenticated API calls. Code only.
   ```

   When you submit this prompt, Claude Code:
   1. Loads the Agent Auth skill from the marketplace
   2. Analyzes your existing application structure
   3. Generates Scalekit client initialization
   4. Creates connected account management functions
   5. Implements OAuth authorization link generation
   6. Adds token fetching and refresh logic

   <Aside type="caution" title="Review generated code">
   Always review AI-generated authentication code before deployment. Verify that environment variables, token validation logic, and error handling match your security requirements. The coding agent provides a foundation, but you must ensure it aligns with your application's specific needs.
   </Aside>

5. ## Verify and test the implementation

   After the coding agent completes, verify that all authentication components are properly configured:

   Check generated files:
   - Scalekit client initialization with credentials
   - Connected account management functions
   - Authorization link generation
   - Token fetching and storage
   - Error handling for expired tokens

   **Test the authentication flow:**

   <Tabs syncKey="tech-stack">
   <TabItem value="claude" label="Claude Code">
   ```md wrap title="Claude REPL" showLineNumbers=false
   Now that your Agent Auth integration is complete, let's verify it's working correctly by testing the flow step by step. First, set your environment variables for SCALEKIT_CLIENT_ID, SCALEKIT_CLIENT_SECRET, and SCALEKIT_ENV_URL. Next, run your application to create a connected account for a test user and generate an authorization link. Open the authorization link in your browser and complete the OAuth consent flow to grant access to your Gmail account. Once authorized, use the Scalekit SDK to fetch the access token and test making a simple API call to Gmail, such as fetching your latest emails. If the emails are returned successfully, your Agent Auth implementation is functioning correctly and your AI agent can now take actions on behalf of your users.
   ```
   </TabItem>
   <TabItem value="node" label="Node.js">
   ```bash title="Terminal" frame="terminal"
   # Set environment variables
   export SCALEKIT_CLIENT_ID=your_client_id
   export SCALEKIT_CLIENT_SECRET=your_client_secret
   export SCALEKIT_ENV_URL=https://your-env.scalekit.com

   # Run your application
   npm start

   # Follow the authorization link printed in the console
   # Complete the OAuth flow in your browser

   # Verify the application can fetch authenticated data
   ```
   </TabItem>
   <TabItem value="python" label="Python">
   ```bash title="Terminal" frame="terminal"
   # Set environment variables
   export SCALEKIT_CLIENT_ID=your_client_id
   export SCALEKIT_CLIENT_SECRET=your_client_secret
   export SCALEKIT_ENV_URL=https://your-env.scalekit.com

   # Run your application
   python main.py

   # Follow the authorization link printed in the console
   # Complete the OAuth flow in your browser

   # Verify the application can fetch authenticated data
   ```
   </TabItem>
   </Tabs>

   The authorization flow should redirect users to the service's consent page, where they grant permissions. Your application should then be able to fetch OAuth tokens and execute actions on behalf of the authenticated user.

</Steps>

## Next steps

Your AI application now has Agent Auth integrated. Test the implementation with your AI agent to verify it can successfully execute actions on behalf of authenticated users.

### Test with AI frameworks

   Connect your Agent Auth integration to popular AI frameworks:

   - **LangChain**: Use Scalekit's LangChain tools to give agents access to external services
   - **Google ADK**: Build agents with Gemini models and Agent Auth
   - **MCP servers**: Create MCP servers that expose tools from authenticated services
   - **Custom frameworks**: Use Scalekit's SDKs to integrate with any AI framework

When you connect, the agent authenticates users through the OAuth 2.0 flow you configured. Verify that protected resources require valid access tokens and that the agent can successfully execute actions on behalf of authenticated users.
