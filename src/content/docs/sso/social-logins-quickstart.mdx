Single sign-on (SSO) lets users authenticate via their organization's identity provider (IdP). Many teams also use social IdPs such as Google, Microsoft, and GitHub.

This guide shows you how to add social login to your B2B SaaS using Scalekit and OAuth 2.0.

  ![How Scalekit works](@/assets/docs/common/social-connections/0.png)

<Steps>

1. ## Set up Scalekit and register a callback endpoint

   Follow the [installation guide](/authenticate/installation/) to configure Scalekit in your application.

   Scalekit verifies user identities and creates sessions. After successful verification, Scalekit creates a user record and sends the user information to your callback endpoint.

   **Create a callback endpoint:**

   1. Add a callback endpoint to your application (typically `https://your-app.com/auth/callback`)
   2. Register this URL in your Scalekit dashboard

   Learn more about [callback URL requirements](/guides/dashboard/redirects/#allowed-callback-urls).

3. ## Create the authorization URL

   Redirect users to Scalekit's authorization endpoint with the chosen social provider.

   Required parameters:

    | Parameter | Description |
    | :-------- | :---------- |
    | `redirect_uri` | Your application's callback URL to receive the authorization `code`. |
    | `client_id` | Your Scalekit application identifier (per environment). |
    | `provider` | The provider code (for example, `google`, `microsoft`). |

    ```sh title="Example authorization URL"
    https://auth.scalekit.com/authorize?
      client_id=skc_122056050118122349527&
      redirect_uri=https://yourapp.com/auth/callback&
      provider=google
    ```
